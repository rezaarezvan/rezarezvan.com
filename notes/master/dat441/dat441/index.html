Introduction to Multi-Armed Bandits

Multi-armed bandits 1 are a simple but very powerful framework for sequential decision making over time under uncertainty. We can also view them as a variant of online learning 2, where the problem instances arrive sequentially in an online manner.

It is an interactive machine learning paradigm between an agent and the environment. The task is to find a principled way to balance the tradeoff between explotation and exploration, which we will define later.

Basic Example

We have put a rat in a cage with two levers, both of which are stochastic. When our agent (rat) pulls a lever, it receives a reward drawn from a fixed but unknown distribution associated with that lever. For simplicity, we have two possible rewards: 0 (electrical shock) and 1 (piece of cheese).

So given our current information, logically we would want to pull the lever that has given us the most cheese in the past.

Seems like we need to explore more to get more information about the levers.

This is just a naïve example, we can have,

Example: More Examples
  • News website: When a new users arrives,
    • A website picks an article header to show.
    • It observes whether the user clicks on this header.
    • Thus, the site’s goal is to maximize the total number of clicks.
  • Dynamic pricing: A store is selling a digital good, e.g., an app or a song. When a new customer arrives,
    • The store chooses a price offered to this customer.
    • The customer buys (or not) the good and leaves forever.
    • The store’s goal is to maximize the total profit.
  • Investment: Each morning,
    • You choose one stock to invest into and invest $1.
    • At the end of the day, you observe the change in value for each stock.
    • The goal is to maximize the total return.

One important aspect of these problems is that the best long-term strategy may involve short-term sacrifices. We want to gather (enough) information to make the best overall decisions.

Basic Setup and Notation

Multi-armed bandit unifies these above examples (and many more).

In the basic version, our agent has K\color{blue}{K}K possible actions to choose from, which are called arms\color{blue}{\textbf{arms}}arms, and our agent has T\color{green}{T}T (time steps/epochs)\color{green}{\text{(time steps/epochs)}}(time steps/epochs) rounds to make decisions.

During each round\color{green}{\text{round}}round, our agent chooses an arm\color{red}{\text{chooses an arm}}chooses an arm and collects a reward for this arm\color{red}{\text{collects a reward for this arm}}collects a reward for this arm.

The reward is drawn independently from some distribution which is fixed (i.e., depends only on the chosen arm), but not known to our agent.

Explotation & Exploration

In this basic setup, our agent observes the reward for the chosen arm after each round, but not for the other arms that could have been chosen.

Therefore, our agent needs to explore, try out different arms to acquire more (and new) information. Thus, we have a tradeoff between exploration and explotation, making (optimal) near-term decisions based on the available information.

Essentially, our agent strives to learn which arm(s) are the best while not spending too much time exploring (suboptimal arms).

Concentration Inequalities

Before we proceed with more bandits, we need to introduce some mathematical background.

Consider nnn random variables X1,X2,…,XnX_1, X_2, \ldots, X_nX1​,X2​,…,Xn​, that are mutually independent but not necessarily identically distributed.

Let Xˉn=X1+…+Xnn\bar{X}_n = \frac{X_1 + \ldots + X_n}{n}Xˉn​=nX1​+…+Xn​​ be their empirical mean with the expectation μn=E[Xˉn]\mu_n = \mathbb{E}[\bar{X}_n]μn​=E[Xˉn​].

According to the Strong Law of Large Numbers 3,

P(Xˉn→μn)=1with n→∞.P(\bar{X}_n \to \mu_n) = 1 \quad \text{with } n \to \infty.P(Xˉn​→μn​)=1with n→∞.

An interesting question is “how concentratred Xˉn\bar{X}_nXˉn​ is around μn\mu_nμn​?“a

As we can see from the above figures, we need some kind of measurement to quantify the concentration of Xˉn\bar{X}_nXˉn​ around μn\mu_nμn​.

In other words, is Xˉn−μn\bar{X}_n - \mu_nXˉn​−μn​ small with high probability?

P[∣Xˉn−μn∣<ϵ]≥1−δ,for some small ϵ,δ>0P[|\bar{X}_n - \mu_n| < \epsilon] \geq 1 - \delta, \quad \text{for some small } \epsilon, \delta > 0P[∣Xˉn​−μn​∣<ϵ]≥1−δ,for some small ϵ,δ>0

To understand this, we will introduce confidence radius (and intervals).

For a fixed nnn, we consider the following high-probability event 1This equation might seem arbitrary, but this comes from Hoeffding’s Inequality, which we will cover soon.,

ϵα,β≔∣Xˉn−μn∣≤αβlog⁡(T)n,α>0.\begin{equation} \epsilon_{\alpha, \beta} \coloneqq |\bar{X}_n - \mu_n| \leq \sqrt{\frac{\alpha \beta \log(T)}{n}}, \quad \alpha > 0. \end{equation}ϵα,β​:=∣Xˉn​−μn​∣≤nαβlog(T)​​,α>0.​​

Then, the following statement holds under various assumptions,

P(ϵα,β)≥1−2T−2α,∀α>0.\begin{equation} P(\epsilon_{\alpha, \beta}) \geq 1 - 2 T^{-2\alpha}, \forall \alpha > 0. \end{equation}P(ϵα,β​)≥1−2T−2α,∀α>0.​​

In the above equations, TTT is a fixed parameter, e.g., the time horizon in the multi-armed bandit problem. The parameter α\alphaα controls the failure probability, we will usually set this to 2. β\betaβ depends on the problem assumptions (often fixed at 1).

We can thus define confidence radius as follows,

rn=αlog⁡(T)n,r_n = \sqrt{\frac{\alpha \log(T)}{n}},rn​=nαlog(T)​​,

and the confidence interval is [μn−rn,μn+rn][\mu_n - r_n, \mu_n + r_n][μn​−rn​,μn​+rn​].

Stochastic Bandits

We now will consider the basic stochastic bandits model, with the set of KKK arms A\mathcal{A}A, with rewards being i.i.d.

FOr each arm/action aaa there is a distribution Da\mathcal{D}_aDa​ over reals, called reward distribution, which are initially unknown.

For simplicity, we will assume that our rewards are bounded in [0,1][0, 1][0,1].

The thing of interest is the mean reward vector μ∈[0,1]K\mu \in [0, 1]^Kμ∈[0,1]K, where μ(a)≔E[Da]\mu(a) \coloneqq \mathbb{E}[\mathcal{D}_a]μ(a):=E[Da​] is the mean reward of arm aaa.

Example

Consider a Bernoulli distribution, where the reward of each arm aaa can be either 1 or 0 (“success” or failure”, “heads or tails”, etc.).

This reward distribution is fully specified by the mean reward, the probability of the successful outcome.

Regret

We know that the best mean reward is μ⋆≔max⁡a∈Aμ(a)\mu^{\star} \coloneqq \underset{a \in \mathcal{A}}{\max} \mu(a)μ⋆:=a∈Amax​μ(a).

We can define the gap of an arm aaa as Δ(a)≔μ⋆−μ(a)\Delta(a) \coloneqq \mu^{\star} - \mu(a)Δ(a):=μ⋆−μ(a).

Thus, the optimal arm in an arm with μ(a)=μ⋆\mu(a) = \mu^{\star}μ(a)=μ⋆.

We usually use a⋆a^{\star}a⋆ to denote some optimal arm (note, this is not necessarily unique).

With these, we can define the regret (at round T),

R(T)=μ⋆⋅T−∑t=1Tμ(at)R(T) = \mu^{\star} \cdot T - \sum_{t=1}^T \mu(a_t)R(T)=μ⋆⋅T−t=1∑T​μ(at​)

where ata_tat​ is the arm chosen at round ttt, is a random quantity that depends on the randomness in rewards/agent. Thus, R(T)R(T)R(T) is a random variable that compares the agent’s cumulative reward to the best arm benchmark μ⋆⋅T\mu^{\star} \cdot Tμ⋆⋅T.

Since R(T)R(T)R(T) is a random variable, we are interested about the expected regret E[R(T)]\mathbb{E}[R(T)]E[R(T)].

Explore-First Algorithm

We will now introduce our first (simple) algorithm to the bandit problem, called Explore-First.

This algorithm is based on a uniform exploration strategy, we dedicate an initial segment of rounds to exploration, and the remaining rounds to exploitation.

Analysis of Explore-First

We will now analyze the expected regret of this algorithm.

Let μˉ(a)\bar{\mu}(a)μˉ​(a) be the average reward for each arm aaa after the exploration phase. Thus, μˉ(a)\bar{\mu}(a)μˉ​(a) is desired to be close to μ(a)\mu(a)μ(a), i.e., ∣μˉ(a)−μ(a)∣|\bar{\mu}(a) - \mu(a)|∣μˉ​(a)−μ(a)∣ is small.

We can apply Hoeffding’s Inequality (with α=2,β=1\alpha = 2, \beta = 1α=2,β=1) to get,

P(∣μˉ(a)−μ(a)∣≤r(a))≥1−2T−4,r(a)=4log⁡(T)n.\begin{equation} P(|\bar{\mu}(a) - \mu(a)| \leq r(a)) \geq 1 - 2 T^{-4}, \quad r(a) = \sqrt{\frac{4 \log(T)}{n}}. \end{equation}P(∣μˉ​(a)−μ(a)∣≤r(a))≥1−2T−4,r(a)=n4log(T)​​.​​

The probability that the average will deviate from the true expectation is very small.

But let’s analyze further, for this we will need to define clean and bad events.

Clean & Bad Events (K=2K = 2K=2)

We define a clean event as the event where Equation (3) holds for all arms simultaneously. Thus, a bad event is the complement of a clean event (i.e., when Equation (3) does not hold for at least one arm).

Let us start with a small example to understand this better. Let K=2K = 2K=2 and A={a,a′}\mathcal{A} = \{a, a^{\prime}\}A={a,a′}.

By definition, the clean event will be,

P(clean)=P(clean(a) and clean(a′)).P(\text{clean}) = P(\text{clean}(a) \text{ and } \text{clean}(a^{\prime})).P(clean)=P(clean(a) and clean(a′)).

From Equation (3), we have,

P(clean(a))≥1−2T−4,P(\text{clean}(a)) \geq 1 - 2 T^{-4},P(clean(a))≥1−2T−4,

Thus,

P(bad(a))≤2T−4.P(\text{bad}(a)) \leq 2 T^{-4}.P(bad(a))≤2T−4.

By the union bound 4, we have,

P(bad)=P(bad(a) or bad(a′))≤P(bad(a))+P(bad(a′))≤2T−4+2T−4=4T−4.\begin{align*} P(\text{bad}) & = P(\text{bad}(a) \text{ or } \text{bad}(a^{\prime})) \newline & \leq P(\text{bad}(a)) + P(\text{bad}(a^{\prime})) \leq 2 T^{-4} + 2 T^{-4} = 4 T^{-4}. \end{align*}P(bad)​=P(bad(a) or bad(a′))≤P(bad(a))+P(bad(a′))≤2T−4+2T−4=4T−4.​

Which means,

P(clean)=1−4T−4.P(\text{clean}) = 1 - 4 T^{-4}.P(clean)=1−4T−4.

Further, let’s now consider that the best arm is a⋆a^{\star}a⋆, and suppose our agent chooses the other arm a≠a⋆a \neq a^{\star}a=a⋆ (otherwise, there would be no regret).

This means μˉ(a)>μˉ(a⋆)\bar{\mu}(a) > \bar{\mu}(a^{\star})μˉ​(a)>μˉ​(a⋆) 2This means that, after the exploration phase, the empirical mean of the wrong arm is larger, essentially our agent has not “converged” or found the optimal arm yet..

μ(a)+r(a)≥μˉ(a)>μˉ(a⋆)≥μ(a⋆)−r(a⋆)\mu(a) + r(a) \geq \bar{\mu}(a) > \bar{\mu}(a^{\star}) \geq \mu(a^{\star}) - r(a^{\star})μ(a)+r(a)≥μˉ​(a)>μˉ​(a⋆)≥μ(a⋆)−r(a⋆)

From the clean event for both arms, we have,

(clean event for a)−r(a)≤μˉ(a)−μ(a)≤r(a)⇒ μˉ(a)≤μ(a)+r(a)(choice in exploitation)μˉ(a)>μˉ(a⋆)(clean event for a⋆)−r(a⋆)≤μˉ(a⋆)−μ(a⋆)≤r(a⋆)⇒ μˉ(a⋆)≥μ(a⋆)−r(a⋆)combine all threeμ(a)+r(a) ≥ μˉ(a) > μˉ(a⋆) ≥ μ(a⋆)−r(a⋆).\begin{aligned} &\text{(clean event for }a\text{)} && -r(a)\le \bar\mu(a)-\mu(a)\le r(a) \newline & && \Rightarrow\ \bar\mu(a)\le \mu(a)+r(a) \newline &\text{(choice in exploitation)} && \bar\mu(a)>\bar\mu(a^\star) \newline &\text{(clean event for }a^\star\text{)} && -r(a^\star)\le \bar\mu(a^\star)-\mu(a^\star) \le r(a^\star) \newline & && \Rightarrow\ \bar\mu(a^\star)\ge \mu(a^\star)-r(a^\star) \newline &\text{combine all three} && \mu(a)+r(a)\ \ge\ \bar\mu(a)\ >\ \bar\mu(a^\star)\ \ge\ \mu(a^\star)-r(a^\star). \end{aligned}​(clean event for a)(choice in exploitation)(clean event for a⋆)combine all three​​−r(a)≤μˉ​(a)−μ(a)≤r(a)⇒ μˉ​(a)≤μ(a)+r(a)μˉ​(a)>μˉ​(a⋆)−r(a⋆)≤μˉ​(a⋆)−μ(a⋆)≤r(a⋆)⇒ μˉ​(a⋆)≥μ(a⋆)−r(a⋆)μ(a)+r(a) ≥ μˉ​(a) > μˉ​(a⋆) ≥ μ(a⋆)−r(a⋆).​

Thus, we have,

μ(a⋆)−μ(a)≤r(a)+r(a⋆)=O(log⁡(T)n).\mu(a^{\star}) - \mu(a) \leq r(a) + r(a^{\star}) = \mathcal{O}\left(\sqrt{\frac{\log(T)}{n}}\right).μ(a⋆)−μ(a)≤r(a)+r(a⋆)=O(nlog(T)​​).

Which means that, for each round in the exploitation phase contributes at most O(log⁡(T)n)\mathcal{O}\left(\sqrt{\frac{\log(T)}{n}}\right)O(nlog(T)​​) to the regret.

And each round in the exploration phase contributes at most 1 to the regret 3This is because the rewards are bounded between 0 and 1 in our case, so always pulling a suboptimal arm can incur at most 1 regret per round..

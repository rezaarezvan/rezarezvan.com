Bayesian Bandit Problem

So far, we have discussed the stochastic bandits model, with a set of KKK arms A\mathcal{A}A.

In the Bayesian bandit problem, we add the Bayesian assumption.

Definition: Bayesian Bandit Problem

The Bayesian bandit problem is a stochastic bandit problem where the problem instance J\mathcal{J}J is drawn initially from sone known distribution P\mathbb{P}P.

The problem instance J\mathcal{J}J (often) specifies the mean reward vector μ∈[0,1]K\mathbf{\mu} \in [0, 1]^Kμ∈[0,1]K.

The distribution P\mathbb{P}P is called the prior distribution, or the Bayesian prior.

As we saw for normal stochastic bandits, the objective is to minimize the expected regret, thus, in the Bayesian setting, we want to minimize the Bayesian expected regret.

Definition: Bayesian Expected Regret

The Bayesian Regret is the expected regret for a particular problem instance J\mathcal{J}J, in expectation over the problem instances,

BR(T)≔EJ∼P[E[R(T)∣J]]=EJ∼P[E[μ⋆⋅T−∑t∈[T]μ(at)]]\mathrm{BR}(T) \coloneqq \mathbb{E}_{\mathcal{J} \sim \mathbb{P}}[\mathbb{E}[R(T) \mid \mathcal{J}]] = \mathbb{E}_{\mathcal{J} \sim \mathbb{P}}\left[ \mathbb{E}\left[ \mu^{\star} \cdot T - \sum_{t \in [T]} \mu(a_t) \right] \right]BR(T):=EJ∼P​[E[R(T)∣J]]=EJ∼P​​E​μ⋆⋅T−t∈[T]∑​μ(at​)​​

Further, we will assume that our realized rewards are drawn from a single-parameter family of distributions (e.g., Bernoulli, unit-variance Gaussian, etc.). Thus, the problem instance is completely specified by the mean reward vector μ∈[0,1]K\mathbf{\mu} \in [0, 1]^Kμ∈[0,1]K.

The prior P\mathbb{P}P is simply a distribution over [0,1]K[0, 1]^K[0,1]K that the μ(⋅)\mu(\cdot)μ(⋅) is drawn from.

Furthermore, we will assume that our realized rewards can only take finitely many different values, and the prior P\mathbb{P}P has a finite support, F\mathcal{F}F.

Lastly, for simplicity, we assume that the best arm a⋆a^{\star}a⋆ is unique for each mean reward vector in the support of P\mathbb{P}P.

We will introduce some notation that we will use later for our analysis. The concepts themselves are not new, but we introduce the compact notation to make our analysis easier.

Posterior As A New Prior

The posterior PH\mathbb{P}_HPH​ can be used as a prior for a subsequent Bayesian update.

Definition: Posterior as a New Prior

Consider a feasible (t+t′)(t + t^{\prime})(t+t′)-history. We represent it as a concatenation of feasible ttt-history HHH and feasible t′t^{\prime}t′-history H′H^{\prime}H′, where HHH comes first.

We can denote the concatenation as H⊕H′H \oplus H^{\prime}H⊕H′.

From this, one can perform the Bayesian update and obtain the posterior (PH)H′(\mathbb{P}_H)_{H^{\prime}}(PH​)H′​ in two steps,

  1. Condition on HHH and derive the posterior PH\mathbb{P}_HPH​, then,

  2. Condition on H′H^{\prime}H′ using PH\mathbb{P}_HPH​ as the new prior to obtain (PH)H′(\mathbb{P}_H)_{H^{\prime}}(PH​)H′​.

Independent Priors

The prior P\mathbb{P}P is called independent if (μ(a):a∈A)(\mu(a) : a \in \mathcal{A})(μ(a):a∈A) are mutually independent random variables.

This essentially means that our update is even simpler, since we can update each arm’s mean reward distribution separately.

Computational Aspects

With brute-force computation of the posterior PH\mathbb{P}_HPH​ at each round ttt, the runtime is O(t⋅∣F∣)\mathcal{O}(t \cdot |\mathcal{F}|)O(t⋅∣F∣), which is impractical for large ttt.

With sequential (Bayesian) updates, the per-round running time is O(∣F∣)\mathcal{O}(|\mathcal{F}|)O(∣F∣), which is much more manageable.

With independent priors, one needs to update only the played arm at each round.

However, with conjugate priors 2, the posterior can be computed directly without summation over all elements in the support F\mathcal{F}F.

Thompson Sampling

This is the idea behind the famous Thompson Sampling algorithm.

The idea is that, for each round ttt and for each arm aaa, our agent computes the posterier probability that aaa is the best arm, and samples aaa with this probability.

Summary: Thompson Sampling Algorithm

For each round t∈[T]t \in [T]t∈[T], do,

  • Observe Ht−1=HH_{t - 1} = HHt−1​=H, for some feasible (t−1)(t - 1)(t−1)-history HHH.
  • Sample mean reward vector μt\mu_tμt​ from the posterior distribution PH\mathbb{P}_HPH​.
  • Choose the best arm a~t\tilde{a}_ta~t​ according to μt\mu_tμt​, i.e., a~t∈arg⁡max⁡a∈Aμt(a)\tilde{a}_t \in \arg\max_{a \in \mathcal{A}} \mu_t(a)a~t​∈argmaxa∈A​μt​(a).

Case Study

A robot is in need of recharging, there are three different power sockets. When tried, each socket randomly returns some charge or gives no charge at all.

We can model the reward from each socket as a Bernoulli random variable, either 1 (charge) or 0 (no charge).

The ideal distribution to model this problem is the Beta distribution, which is a conjugate prior for the Bernoulli distribution.

The Beta distribution has two parameters, α\alphaα and β\betaβ. In the simplest terms, these parameters respectively corresponds to the count of successes and failures (or, more precisely, α−1\alpha - 1α−1 and β−1\beta - 1β−1).

Initially, we have no idea what the probability is of any given socket producing an output, so we can start by setting both α\alphaα and β\betaβ to 1, which corresponds to a uniform distribution over [0,1][0, 1][0,1].

This initial guess at the probability of the socket producing an output corresponds to choosing the prior distribution.

Once the robot tests a socket and obtains a reward, it can modify its belief in the success rate of that socket, i.e., updates the posterior distribution.

The Beta distribution is a conjugate prior for the Bernoulli distribution, so the posterior distribution is also a Beta distribution, with updated parameters.

If the tested socket returns a reward of 1 (success), we increment α\alphaα by 1 and leave β\betaβ unchanged. If the tested socket returns a reward of 0 (failure), we increment β\betaβ by 1 and leave α\alphaα unchanged.

Since actions that have been tried infrequently have wide distributions, they have a larger range of possible values. In this way, a socket that currently has a low estimated mean reward, but has been tested fewer times than a socket with a higher estimated mean, can return a larger sample value and therefore become the selected socket at this time step.

We can now state the Bayesian regret bound for Thompson Sampling.

BR(T)≤O(KTlog⁡(T))\mathrm{BR}(T) \leq \mathcal{O}\left( \sqrt{K T \log(T)} \right)BR(T)≤O(KTlog(T)​)

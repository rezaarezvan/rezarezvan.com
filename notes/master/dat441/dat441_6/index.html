Introduction to Dynamic Programming

Dynamic programming (DP) is a collection of algorithms that can be used to compute optimal policies given a perfect model of the environment (the dynamics p\mathcal{p}p), as a Markov Decision Process (MDP).

We will only consider an environment where the MDP is finite, i.e., the sets of states, actions, and rewards (S,A,R\mathcal{S}, \mathcal{A}, \mathcal{R}S,A,R) all have a finite number of elements and the dynamics are given by a set of probabilities p(s′,r∣s,a)\mathcal{p}(s^{\prime}, r | s, a)p(s′,r∣s,a) for all s∈S,a∈A(s),r∈R,s′∈S+s \in \mathcal{S}, a \in \mathcal{A}(s), r \in \mathcal{R}, s^{\prime} \in \mathcal{S}^+s∈S,a∈A(s),r∈R,s′∈S+.

Policy Evaluation

Firstly, recall that we can easily obtain optimal policies once we have found the optimal value functions, v⋆v_{\star}v⋆​ or q⋆q_{\star}q⋆​, which satisfy the Bellman optimality equations,

Theorem 1 (Bellman Optimality Equation for v⋆v_{\star}v⋆​)v⋆(s)=max⁡aE[Rt+1+γv⋆(St+1)∣St=s,At=a]=max⁡a∑s′,rp(s′,r∣s,a)[r+γv⋆(s′)],q⋆(s,a)=E[Rt+1+γmax⁡a′q⋆(St+1,a′)∣St=s,At=a]=∑s′,rp(s′,r∣s,a)[r+γmax⁡a′q⋆(s′,a′)],\begin{align*} v_{\star}(s) & = \max_a \mathbb{E}[R_{t+1} + \gamma v_{\star}(S_{t+1}) | S_t = s, A_t = a] \newline & = \max_a \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_{\star}(s^{\prime})], \newline \newline q_{\star}(s, a) & = \mathbb{E}[R_{t+1} + \gamma \max_{a^{\prime}} q_{\star}(S_{t+1}, a^{\prime}) | S_t = s, A_t = a] \newline & = \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma \max_{a^{\prime}} q_{\star}(s^{\prime}, a^{\prime})], \newline \end{align*}v⋆​(s)q⋆​(s,a)​=amax​E[Rt+1​+γv⋆​(St+1​)∣St​=s,At​=a]=amax​s′,r∑​p(s′,r∣s,a)[r+γv⋆​(s′)],=E[Rt+1​+γa′max​q⋆​(St+1​,a′)∣St​=s,At​=a]=s′,r∑​p(s′,r∣s,a)[r+γa′max​q⋆​(s′,a′)],​

Now consider that we compute the state-value function vπv_{\pi}vπ​ for an arbitrary policy π\piπ,

vπ(s)=Eπ,p[Gt∣St=s]=Eπ,p[Rt+1+γGt+1∣St=s]=Eπ,p[Rt+1+γvπ(St+1)∣St=s]=∑aπ(a∣s)∑s′,rp(s′,r∣s,a)[r+γvπ(s′)]\begin{align*} v_{\pi}(s) & = \mathbb{E}_{\pi, \mathcal{p}}[G_t | S_t = s] \newline & = \mathbb{E}_{\pi, \mathcal{p}}[R_{t+1} + \gamma G_{t+1} | S_t = s] \newline & = \mathbb{E}_{\pi, \mathcal{p}}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) | S_t = s] \newline & = \sum_a \pi(a | s) \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_{\pi}(s^{\prime})] \end{align*}vπ​(s)​=Eπ,p​[Gt​∣St​=s]=Eπ,p​[Rt+1​+γGt+1​∣St​=s]=Eπ,p​[Rt+1​+γvπ​(St+1​)∣St​=s]=a∑​π(a∣s)s′,r∑​p(s′,r∣s,a)[r+γvπ​(s′)]​
Note

With completely known environment dynamics p\mathcal{p}p, the above equation is a system of ∣S∣|\mathcal{S}|∣S∣ linear equations of vπ,s∈Sv_{\pi}, s \in \mathcal{S}vπ​,s∈S, unknowns. Thus, it can be solved directly by standard methods for solving linear equations (e.g., Gaussian elimination). However, this is often computationally expensive and impractical for large state spaces.

Iterative Policy Evaluation

Thus, we instead use iterative methods that are more computationally efficient.

Definition 1 (Iterative Policy Evaluation)

Initially approximate the state values arbitrarily, e.g., v0(s)=0v_0(s) = 0v0​(s)=0 for all s∈Ss \in \mathcal{S}s∈S.

Then, update the approximations by using the Bellman equation for vπv_{\pi}vπ​,

vk+1(s)≔Eπ,p[Rt+1+γvk(St+1)∣St=s]=∑aπ(a∣s)∑s′,rp(s′,r∣s,a)[r+γvk(s′)]\begin{align*} v_{k + 1}(s) & \coloneqq \mathbb{E}_{\pi, \mathcal{p}}[R_{t+1} + \gamma v_k(S_{t+1}) | S_t = s] \newline & = \sum_a \pi(a | s) \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_k(s^{\prime})] \end{align*}vk+1​(s)​:=Eπ,p​[Rt+1​+γvk​(St+1​)∣St​=s]=a∑​π(a∣s)s′,r∑​p(s′,r∣s,a)[r+γvk​(s′)]​
Note

The sequence {vk}\{v_k\}{vk​} can be shown to converge to vπv_{\pi}vπ​ as k→∞k \to \inftyk→∞.

Policy Improvement

But how do we use our newly computed value function vπv_{\pi}vπ​ to find a better policy?

Consider a one-step lookahead, i.e., we select a∈A(s)a \in \mathcal{A}(s)a∈A(s) that maximizes the expected return by taking action aaa in state sss and thereafter following policy π\piπ.

The value of this one-step lookahead is,

qπ(s,a)=E[Rt+1+γvπ(St+1)∣St=s,At=a]=∑s′,rp(s′,r∣s,a)[r+γvπ(s′)]\begin{align*} q_{\pi}(s, a) & = \mathbb{E}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) | S_t = s, A_t = a] \newline & = \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_{\pi}(s^{\prime})] \newline \end{align*}qπ​(s,a)​=E[Rt+1​+γvπ​(St+1​)∣St​=s,At​=a]=s′,r∑​p(s′,r∣s,a)[r+γvπ​(s′)]​

If we assume that qπ(s,a)q_{\pi}(s, a)qπ​(s,a) is greater than vπ(s)v_{\pi}(s)vπ​(s), then we may expect that taking action aaa in state sss and thereafter following policy π\piπ will yield a greater return than simply following policy π\piπ from the start, thus yielding an overall better policy.

This is a special cae of a general result called the policy improvement theorem.

Policy Improvement Theorem

Theorem 2 (Policy Improvement Theorem)

Let π\piπ and π′\pi^{\prime}π′ be any pair of deterministic policies such that, for all s∈Ss \in \mathcal{S}s∈S,

qπ(s,π′(s))≥vπ(s)q_{\pi}(s, \pi^{\prime}(s)) \geq v_{\pi}(s)qπ​(s,π′(s))≥vπ​(s)

Then, the policy π′\pi^{\prime}π′ must be as good as or better than policy π\piπ, i.e., it must obtain greater or equal return from all states s∈Ss \in \mathcal{S}s∈S,

vπ′(s)≥vπ(s)v_{\pi^{\prime}}(s) \geq v_{\pi}(s)vπ′​(s)≥vπ​(s)

We will now prove this theorem.

Proof (Policy Improvement Theorem)vπ(s)≤qπ(s,π′(s))=E[Rt+1+γvπ(St+1)∣St=s,At=π′(s)]=Eπ′[Rt+1+γvπ(St+1)∣St=s]≤Eπ′[Rt+1+γqπ(St+1,π′(St+1))∣St=s]=Eπ′[Rt+1+γEπ′[Rt+2+γvπ(St+2)∣St+1]∣St=s]=Eπ′[Rt+1+γRt+2+γ2vπ(St+2)∣St=s]≤Eπ′[Rt+1+γRt+2+γ2Rt+3+γ3vπ(St+3)∣St=s]≤…=vπ′(s)\begin{align*} v_{\pi}(s) & \leq q_{\pi}(s, \pi^{\prime}(s)) \newline & = \mathbb{E}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) | S_t = s, A_t = \pi^{\prime}(s)] \newline & = \mathbb{E}_{\pi^{\prime}}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) | S_t = s] \newline & \leq \mathbb{E}_{\pi^{\prime}}[R_{t+1} + \gamma q_{\pi}(S_{t+1}, \pi^{\prime}(S_{t+1})) | S_t = s] \newline & = \mathbb{E}_{\pi^{\prime}}[R_{t+1} + \gamma \mathbb{E}_{\pi^{\prime}}[R_{t+2} + \gamma v_{\pi}(S_{t+2}) | S_{t+1}] | S_t = s] \newline & = \mathbb{E}_{\pi^{\prime}}[R_{t+1} + \gamma R_{t+2} + \gamma^2 v_{\pi}(S_{t+2}) | S_t = s] \newline & \leq \mathbb{E}_{\pi^{\prime}}[R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + \gamma^3 v_{\pi}(S_{t+3}) | S_t = s] \newline & \leq \ldots \newline & = v_{\pi^{\prime}}(s) \newline \end{align*}vπ​(s)​≤qπ​(s,π′(s))=E[Rt+1​+γvπ​(St+1​)∣St​=s,At​=π′(s)]=Eπ′​[Rt+1​+γvπ​(St+1​)∣St​=s]≤Eπ′​[Rt+1​+γqπ​(St+1​,π′(St+1​))∣St​=s]=Eπ′​[Rt+1​+γEπ′​[Rt+2​+γvπ​(St+2​)∣St+1​]∣St​=s]=Eπ′​[Rt+1​+γRt+2​+γ2vπ​(St+2​)∣St​=s]≤Eπ′​[Rt+1​+γRt+2​+γ2Rt+3​+γ3vπ​(St+3​)∣St​=s]≤…=vπ′​(s)​

Now, we can consider changes at all states, selecting at each state the action that appears best according to qπ(s,a)q_{\pi}(s, a)qπ​(s,a), i.e., the greedy policy with respect to qπq_{\pi}qπ​,

π′(s)≔arg⁡max⁡a∈A(s) qπ(s,a)=arg⁡max⁡a∈A(s) E[Rt+1+γvπ(St+1)∣St=s,At=a]=arg⁡max⁡a∈A(s) ∑s′,rp(s′,r∣s,a)[r+γvπ(s′)]\begin{align*} \pi^{\prime}(s) & \coloneqq \underset{a \in \mathcal{A}(s)}{\arg\max} \ q_{\pi}(s, a) \newline & = \underset{a \in \mathcal{A}(s)}{\arg\max} \ \mathbb{E}[R_{t+1} + \gamma v_{\pi}(S_{t+1}) | S_t = s, A_t = a] \newline & = \underset{a \in \mathcal{A}(s)}{\arg\max} \ \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_{\pi}(s^{\prime})] \newline \end{align*}π′(s)​:=a∈A(s)argmax​ qπ​(s,a)=a∈A(s)argmax​ E[Rt+1​+γvπ​(St+1​)∣St​=s,At​=a]=a∈A(s)argmax​ s′,r∑​p(s′,r∣s,a)[r+γvπ​(s′)]​

By construction, the greedy policy meets the conditions of the policy improvement theorem, thus we can conclude that the greedy policy must be as good as or better than the original policy π\piπ.

Suppose that the new greedy policy π′\pi^{\prime}π′ is as good as, but not better than, the old policy π\piπ. Then vπ′(s)=vπ(s)v_{\pi^{\prime}}(s) = v_{\pi}(s)vπ′​(s)=vπ​(s) for all s∈Ss \in \mathcal{S}s∈S,

vπ(s)=max⁡a E[Rt+1+γvπ′(St+1)∣St=s,At=a]=max⁡a ∑s′,rp(s′,r∣s,a)[r+γvπ′(s′)]\begin{align*} v_{\pi}(s) & = \underset{a}{\max} \ \mathbb{E}[R_{t+1} + \gamma v_{\pi^{\prime}}(S_{t+1}) | S_t = s, A_t = a] \newline & = \underset{a}{\max} \ \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_{\pi^{\prime}}(s^{\prime})] \newline \end{align*}vπ​(s)​=amax​ E[Rt+1​+γvπ′​(St+1​)∣St​=s,At​=a]=amax​ s′,r∑​p(s′,r∣s,a)[r+γvπ′​(s′)]​

This is the same as the Bellman optimality equation for v⋆v_{\star}v⋆​, thus vπ=vπ′=v⋆v_{\pi} = v_{\pi^{\prime}} = v_{\star}vπ​=vπ′​=v⋆​, and both policies π\piπ and π′\pi^{\prime}π′ are optimal policies, i.e., π,π′∈{π⋆}\pi, \pi^{\prime} \in \{\pi_{\star}\}π,π′∈{π⋆​}.

Policy Iteration

We can now combine policy evaluation and policy improvement to form the policy iteration algorithm.

Once a policy π\piπ, has been improved using vπv_{\pi}vπ​ to yield a better policy π′\pi^{\prime}π′, we can then compute vπ′v_{\pi^{\prime}}vπ′​ and improve it again to yield an even better policy π′′\pi^{\prime\prime}π′′, and so on.

Each policy is guaranteed to be a strict improvement over the previous one, unless the policy is already optimal, in which case the policy improvement step will yield the same policy again.

Because we assume a finite MDP, only a finite number of policies exist, thus this process must eventually terminate with an optimal policy.

Value Iteration

The policy evaluation step in policy iteration is an iterative procedure that can be computationally expensive.

We can instead truncate the policy evaluation step after only one sweep of updates over all states, i.e., we update each state value V(s)V(s)V(s) only once before moving on to the policy improvement step.

Note that we can truncate in several ways, wihtout losing the guarantee of convergence to the optimal policy.

One extreme case is value iteration; Stop the policy evaluation after just one sweep (one update of each state), and then immediately perform the policy improvement step.

We can further extend the policy improvement step and truncated policy evaluation step into a single update,

vk+1(s)≔max⁡aE[Rt+1+γvk(St+1)∣St=s,At=a]=max⁡a∑s′,rp(s′,r∣s,a)[r+γvk(s′)]\begin{align*} v_{k + 1}(s) & \coloneqq \max_a \mathbb{E}[R_{t+1} + \gamma v_k(S_{t+1}) | S_t = s, A_t = a] \newline & = \max_a \sum_{s^{\prime}, r} \mathcal{p}(s^{\prime}, r | s, a) [r + \gamma v_k(s^{\prime})] \newline \end{align*}vk+1​(s)​:=amax​E[Rt+1​+γvk​(St+1​)∣St​=s,At​=a]=amax​s′,r∑​p(s′,r∣s,a)[r+γvk​(s′)]​

Or in other words, we turn the Bellman optimality equation for v⋆v_{\star}v⋆​ into an update rule.

Variants

Value iteration effectively combines, in each of its sweeps, one sweep of policy evaluation and one sweep of policy improvement. However, we can gain faster convergence with interposing multiple policy evaluation sweeps between each policy improvement sweep.

In general, the entire class of truncated policy iteration algorithms can be thought of as a sequence of sweeps, some of which use policy evaluation updates and some of which use value iteration updates. Note that, because the only difference between the two types of updates is whether we use a max⁡\maxmax, we can even intermix the two types of updates within a single sweep.

Note

All of these algorithms converge to an optimal policy for discounted, finite MDPs.

Asynchronous DP

Performing sweeps over the entire state set can be computationally expensive.

Thus, asynchronous DP algorithms update the values of states in any order whatsoever, using whatever values of other states happen to be available. The values of some states may be updated several times before the values of others are updated even once!

To converge correctly, all states should be updated properly.

Generalized Policy Iteration (GPI)

Policy iteration consists of two simultaneous, interacting proceses.

  • One making the value function consistent with the current policy (policy evaluation), and,
  • The other making the policy greedy with respect to the current value function (policy improvement).

There are several other variants as well.

As long as both processes continue to update all states, the ultimate result is typically the same - convergence to the optimal value function and an optimal policy.

Efficiency of Dynamic Programming

DP may not be practical for very large problems, but compared to other methods for solving MDPs, DP methods are actually quite efficient.

The time that DP methods take to find an optimal policy is polynomial in the number of states and actions.

However, (as always), we are dealing with the curse of dimensionality 1 (i.e., exponential growth of state space), where this might limit the applicability of DP, but it is an inherent difficulty of the problem itself, not of the solution method.

On problems with large state spaces, asynchronous DP methods (and other variations of GPI) are often preferred. To complete even one sweep of a synchronous method requires computation and memory for every state, which may be infeasible.

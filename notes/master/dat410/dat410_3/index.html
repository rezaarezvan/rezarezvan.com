Introduction

When dealing with the machine learning life cycle, we will encounter,

  1. Data collection (Collect user ratings).
  2. Data representation & Processing (Preprocess files, movie_reviews.csv).
  3. Modelling (Collaborative filtering, Content-based filtering).
  4. Learning/Optimization (Minimizing empirical risk of predicting ratings).
  5. Evaluation (Online A/B testing 1).
  6. Deployment (git add . && git commit -m "LGTM" && git push origin master -f).

At each of these steps we use different tools, but first let’s define what an AI tool is.

What is an AI tool?

When we are talking about AI tools, we are not talking about AI-assisted tools, rather, tools that helps us implement AI systems.

The specifics depend (obviously) on the nature of the system.

We will focus on (statistical) machine learning for prototyping in Python.

Data Representation & Processing

In typical machine learning systems, our data is stored in matrix and or vector form, e.g., a classical table of features XXX and labels YYY.

However, more general data usually does not have natural tabular representation. Think of time series, they must cover different time-scales, different lengths and different possible events.

Image data is also not easily stored in (2D) matrices, instead we work with tensors. We usually just think of them as 3D (or higher) matrices, where each depth dimension represents a different color channel.

Graphs are used to represent relations of people, atoms, cities, etc. These can be represented as adjacency matrices.

However (again), despite the different nature of text, images, etc., we can still (usually) store them in matrix and vector form.

But why matrices?

Because (almost) all of machine learning relies on linear algebra,

  1. Linear Regression y=βTx+cy = \beta^T x + cy=βTx+c.
  2. Deep neural networks hl+1=σ(Wl+1hl+bl)h_{l + 1} = \sigma(W_{l + 1} h_l + b_{l})hl+1​=σ(Wl+1​hl​+bl​).
  3. Linear Programming min⁡Ax+bsubject to Bx+c≤0\begin{aligned} \min & \quad Ax + b \newline \text{subject to } & \quad Bx + c \leq 0 \end{aligned}minsubject to ​Ax+bBx+c≤0​.

Matrix and vector operations are fast (enough) and well-studied.

Drawbacks & Data Frames

A major drawback of pure matrix and tensor representations is that columns and rows are anonymous. Sometimes we want to know what each row or column represents (explicitly).

Data frames add indices and names, much like a spreadsheet.

Missing Values

Say that we want to predict cardiovascular risk for patients who visit their general practitioner.

We look at electronic health records (EHR) data containing age, sex, weight and cholesterol levels XXX and the corresponding cardiovascular risk YYY.

When training our machine learning model, inputs are often assumed to be fully observed, but in reality, it is very common for some features to be missing.

Let X~\tilde{X}X~ be the actual observed values (which is a subset of the full feature XXX).

We can represent missing values by a missingness mask MMM.

Thus, we can do,

Xij~={Xijif Mij=0nanotherwise\tilde{X_{ij}} = \begin{cases} X_{ij} & \text{if } M_{ij} = 0 \newline \texttt{nan} & \text{otherwise} \end{cases}Xij​~​={Xij​nan​if Mij​=0otherwise​

Now, suppose we want to learn to predict YYY from our tabular features XXX, what can we do with our missing values in XXX?

There are two common solutions imputation and informative missingness.

Imputation

Attempt to impute missing values,

  1. Reconstruct XXX from X~\tilde{X}X~.
  2. Predict YYY from reconstructed X^\hat{X}X^.

Any method that works (well) for XXX works (well) for our reconstructed X^\hat{X}X^.

Informative Missingness

Make use of missingness itself,

  1. Predict YYY from both X~\tilde{X}X~ and MMM.

From here we can do two more common approaches,

  • Method sensitive to MMM (e.g., XGBoost 2).
  • Simple imputation + missing indicators.

Single VS. Multiple Imputation

The basic idea of imputation is to predict the missing value from observed values of other variables. In the simplest case, we use a single imputation of each value,

f(Age,Sex,Weight)=Cholesterolf(\text{Age}, \text{Sex}, \text{Weight}) = \text{Cholesterol}f(Age,Sex,Weight)=Cholesterol

The imputation functions can be learned by regression on complete observations or observations with less missingness,

f⋆=arg⁡min⁡fE[(f(Age,Sex,Weight)−Cholesterol)2]f^{\star} = \arg \min_{f} \mathbb{E} \left[ \left( f(\text{Age}, \text{Sex}, \text{Weight}) - \text{Cholesterol} \right)^2 \right]f⋆=argfmin​E[(f(Age,Sex,Weight)−Cholesterol)2]

This becomes tricky when all observations have some missing values.

A more popular method for imputation is Multiple Imputation by Chained Equations, or MICE.

Multiple Imputation: Create more than one sample of each missing value to account for variance.

Chained Equations: Impute one value based on imputations of other variables, works without any complete observations.

  1. Give each variable a placeholder imputed value (e.g., mean)
  2. Repeat for a number of iterations
  3. \quad For each variable vvv:
  4. \quad \quad Regress observed vvv on other variables in the dataset (including other imputed values)
  5. \quad \quad Impute missing vvv using regression.

However, imputation methods may fail when data is not missing at random (MAR), i.e., missing values can not be reliably imputed.

A common method in this case is to stitch a simple imputation X^\hat{X}X^ (e.g., 0) and MMM, as binary indicators, together for prediction.

When do we use which method?

  1. Will data be missing at test/use time?
    • → Imputation is not necessary to minimize test error.
  2. Are you fitting a parametric model (e.g., linear regression)?
    • → Imputation becomes important to recover parameters.
  3. Is missingness predictable from observed values (MAR)?
    • → If not, imputation will lead to biased results.

Model Development

A big development patterin in machine learning systems is the fit, predict, score pattern.

This standardizes the common machine learning workflow,

  1. fit(x, y): Train model to e.g., predict yyy from xxx.
  2. predict(x): Predict yyy for xxx.
  3. score(x, y): Evaluate model on data x,yx, yx,y.

fit(x, y)

The function fit(x, y) is responsible for training and storing model parameters that maximize (or minimize) some objective function.

For example, finding the optimal coefficients in OLS or k-NN.

Our arguments x,yx,yx,y may vary depending on our application, in unsupervised learning there is no yyy, but parameters to fit.

predict(x)

The function predict(x) should take a (new) data point xxx and predict the corresponding outcome (e.g., label/cluster) yyy for it.

score(x, y)

The function score(x, y) assigns a score to the prediction made for xxx in comparison to the label yyy.

score(x, y) can also be used for hyperparameter selection, for example using cross-validation.

Data Preprocessing

All algorithms are sensitive to the representation of the input data.

A (trivial) example are the coefficients of OLS, which depend on the scale of the covariates,

y=βT(ax)+c=(aβ)Tx+c.y = \beta^T(ax) + c = (a \beta)^T x + c.y=βT(ax)+c=(aβ)Tx+c.

A common step is to standardize features, i.e., to make them have zero mean and unit variance,

X(i)←X−μ(X)σ(X).X^{(i)} \leftarrow \frac{X - \mu(X)}{\sigma(X)}.X(i)←σ(X)X−μ(X)​.

Formally, μ\muμ and σ\sigmaσ are parameters, they are functions of our training data.

Differentiable Systems

When we fit a machine learning algorithm on a training dataset, we are solving an optimization problem.

Empirical risk minimization (ERM) is the most common optimization problem in machine learning,

min⁡θ1N∑i=1NL(fθ(x(i)),y(i)).\underset{\theta}{\min} \frac{1}{N} \sum_{i = 1}^{N} L(f_{\theta}(x^{(i)}), y^{(i)}).θmin​N1​i=1∑N​L(fθ​(x(i)),y(i)).

Here, LLL is the loss function, fθf_{\theta}fθ​ is the model and θ\thetaθ are the parameters (to be optimized).

Deep learning and many other AI tools are based on ERM in differentiable systems. Systems where the objective function is differentiable in the parameters θ\thetaθ (this means that learning using gradient descent is possible).

Gradient Descent

Consider the linear model f(x)=θTxf(x) = \theta^T xf(x)=θTx of a 1D label y.

We can measure our error using the mean squared error (MSE),

We can use gradient descent to find a (local) minimum. Gradient descent says to move in the direction of −∇R^(θ)-\nabla \hat{R}(\theta)−∇R^(θ),

Here ∇θ\nabla_{\theta}∇θ​ is,

However, gradient descent only has one non-trivial operation, computing the gradient itself.

As long as R^\hat{R}R^ is a composition of differentiable functions of θ\thetaθ, differentiation is easy, this is what we exploit in modern machine learning (backpropagation).

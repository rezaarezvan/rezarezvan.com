Introduction

In this part, we will explore Bayesian linear regression, a probabilistic approach to linear regression that incorporates uncertainty in the model parameters. Firstly, we will recap supervised machine learning, inference, and the problem setting for linear regression.

Supervised Machine Learning

Definition 1 (Supervised Machine Learning)

Given a training set D={(xi,yi)}i=1N\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^{N}D={(xi​,yi​)}i=1N​, where xi\mathbf{x}_ixi​ are free variables (features, covariates, domain points, explanatory variables, etc.) and yiy_iyi​ are the target variables, dependant on xi\mathbf{x}_ixi​ (dependent variables, labels, responses, etc.), the goal of supervised machine learning is to identify an algorithm to predict the label yyy for a new (yet unseen) input x\mathbf{x}x. In other words, learning a model from label data and predicting outputs of new data based on the learned model.w

However, this task is impossible if there is no information about the mechanism relating x\mathbf{x}x and yyy. Thus, we may assume that x\mathbf{x}x and yyy are related via a function y=f~(x)y = \tilde{f}(\mathbf{x})y=f~​(x).

Thus, the goal is to find the best possible approximation of f~(x)\tilde{f}(\mathbf{x})f~​(x), f(x)f(\mathbf{x})f(x) (prediction model), and predict the outcome yyy for x\mathbf{x}x as y^=f(x)\hat{y} = f(\mathbf{x})y^​=f(x).

Again, in more general temrs, we want to identify a predictive algorithm that minimizes the error in the prediction of a new label yyy for an unobserved input x\mathbf{x}x (generalization loss).

As we discussed last time, we have two main approaches, the frequentist and the Bayesian approach. We will rephrase the supervised machine learning problem as a Bayesian inference problem.

The Inference Problem

Definition 2 (The Inference Problem)

From learning, we know that we need to treat x\mathbf{x}x and yyy as random variables. Thus, our objective is predict yyy given the (random) observation of x\mathbf{x}x under the assumption that the joint distribution p(x,y)p(\mathbf{x}, y)p(x,y) is known.

We define the loss function ℓ(y,y^)\ell(y, \hat{y})ℓ(y,y^​) as the cost (loss or risk) incurred when the correct value is yyy while the estimated value is y^\hat{y}y^​. For example, the quadratic loss function is defined as,

ℓ(y,y^)≔(y−y^)2.\ell(y, \hat{y}) \coloneqq (y - \hat{y})^2.ℓ(y,y^​):=(y−y^​)2.

We say there is a optimal prediction y^⋆(x)\hat{y}^{\star}(\mathbf{x})y^​⋆(x) that minimizes the generalization loss (or, generalization error),

Lp(y^)≔Ex,y∼p(x,y)[ℓ(y,y^(x))].L_p(\hat{y}) \coloneqq \mathbb{E}_{\mathsf{\mathbf{x}}, \mathsf{y} \sim p(\mathbf{x}, y)}[\ell(\mathsf{y}, \hat{y}(\mathsf{\mathbf{x}}))].Lp​(y^​):=Ex,y∼p(x,y)​[ℓ(y,y^​(x))].

The solution is “simply”,

y^⋆(x)=arg⁡min⁡y^ Lp(y^)=arg⁡min⁡y^ Ex,y∼p(x,y)[ℓ(y,y^(x))]=arg⁡min⁡y^ Ex∼p(x)[Ey∼p(y∣x)[ℓ(y,y^(x))]]=arg⁡min⁡y^ Ey∼p(y∣x)[ℓ(y,y^(x))]\begin{align*} \hat{y}^{\star}(\mathbf{x}) & = \underset{\hat{y}}{\arg\min} \ L_p(\hat{y}) \newline & = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{x}, \mathsf{y} \sim p(\mathbf{x}, y)}[\ell(\mathsf{y}, \hat{y}(\mathsf{\mathbf{x}}))] \newline & = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{x} \sim p(\mathbf{x})} \left[ \mathbb{E}_{\mathsf{y} \sim p(y \mid \mathsf{\mathbf{x}})}[\ell(\mathsf{y}, \hat{y}(\mathsf{\mathbf{x}}))] \right] \newline & = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{y} \sim p(y \mid \mathbf{x})}[\ell(\mathsf{y}, \hat{y}(\mathbf{x}))] \end{align*}y^​⋆(x)​=y^​argmin​ Lp​(y^​)=y^​argmin​ Ex,y∼p(x,y)​[ℓ(y,y^​(x))]=y^​argmin​ Ex∼p(x)​[Ey∼p(y∣x)​[ℓ(y,y^​(x))]]=y^​argmin​ Ey∼p(y∣x)​[ℓ(y,y^​(x))]​

Thus, the optimal prediction is a function of p(y∣x)p(y \mid \mathbf{x})p(y∣x) and the loss function ℓ(y,y^)\ell(y, \hat{y})ℓ(y,y^​).

For the loss function, ℓ(y,y^)=(y−y^)2\ell(y, \hat{y}) = (y - \hat{y})^2ℓ(y,y^​)=(y−y^​)2,

y^⋆(x)=Ey∼x[y∣x]\hat{y}^{\star}(\mathbf{x}) = \mathbb{E}_{\mathsf{y} \sim \mathbf{x}}[\mathsf{y} \mid \mathbf{x}]y^​⋆(x)=Ey∼x​[y∣x]

i.e., the optimal prediction is the conditional mean of yyy given x\mathbf{x}x.

However, in most scenarios, the joint distribution p(x,y)p(\mathbf{x}, y)p(x,y) is unknown.

Finally, let’s restate the supervised machine learning problem as an inference problem.

The goal is to obtain a predictor y^(x)\hat{y}(\mathbf{x})y^​(x) that performs close to optimal predictor y^⋆(x)\hat{y}^{\star}(\mathbf{x})y^​⋆(x) based only on the training set D\mathcal{D}D (without knowledge of p(x,y)p(\mathbf{x}, y)p(x,y)). Where closeness is measured as,

Lp(y^)−Lp(y^⋆).L_p(\hat{y}) - L_p(\hat{y}^{\star}).Lp​(y^​)−Lp​(y^​⋆).

where Lp(y^)L_p(\hat{y})Lp​(y^​) is the generalization loss of the trained predictor y^(x)\hat{y}(\mathbf{x})y^​(x) and Lp(y^⋆)L_p(\hat{y}^{\star})Lp​(y^​⋆) is the minimum generalization loss (optimal predictor).

Frequentist Approach

Intuition (Frequentist Approach)

Our training data points (xi,yi)∈D(\mathbf{x}_i, \mathsf{y}_i) \in \mathcal{D}(xi​,yi​)∈D are i.i.d random variables drawn from a true (but unknown) distribution p(x,y)p(\mathbf{x}, y)p(x,y),

(xi,yi)∼i.i.dp(x,y),i=1,…,N.(\mathbf{x}_i, \mathsf{y}_i) \sim_{\text{i.i.d}} p(\mathbf{x}, y), \quad i = 1, \ldots, N.(xi​,yi​)∼i.i.d​p(x,y),i=1,…,N.

Since p(x,y)p(\mathbf{x}, y)p(x,y) is unknown, we cannot find the optimal predictor via,

y^⋆(x)=arg⁡min⁡y^ Ey∣x[ℓ(y,y^)∣x].\hat{y}^{\star}(\mathbf{x}) = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{y} \mid \mathbf{x}}[\ell(\mathsf{y}, \hat{y}) \mid \mathbf{x}].y^​⋆(x)=y^​argmin​ Ey∣x​[ℓ(y,y^​)∣x].

The solution however, is to separate learning inference. If we can learn an approximation of p(y∣x)p(y \mid \mathbf{x})p(y∣x) based on D(pD(y∣x))\mathcal{D}(p_{\mathcal{D}}(y \mid \mathbf{x}))D(pD​(y∣x)), and use it in,

y^⋆(x)=arg⁡min⁡y^ Ey∣x[ℓ(y,y^)∣x].\hat{y}^{\star}(\mathbf{x}) = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{y} \mid \mathbf{x}}[\ell(\mathsf{y}, \hat{y}) \mid \mathbf{x}].y^​⋆(x)=y^​argmin​ Ey∣x​[ℓ(y,y^​)∣x].

we can obtain,

y^D(x)=arg⁡min⁡y^ Ey∼pD(y∣x)[ℓ(y,y^)∣x].\hat{y}_{\mathcal{D}}(\mathbf{x}) = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{y} \sim p_{\mathcal{D}}(y \mid \mathbf{x})}[\ell(\mathsf{y}, \hat{y}) \mid \mathbf{x}].y^​D​(x)=y^​argmin​ Ey∼pD​(y∣x)​[ℓ(y,y^​)∣x].

Further, we can perform direct inference via empirical risk minimization (ERM). If we directly learn an approximation of the optimal decision rule (y^D(⋅)\hat{y}_{\mathcal{D}}(\cdot)y^​D​(⋅)) by minimizing an empirical estimate of the generalization loss,

y^D≔arg⁡min⁡y^ LD(y^)=arg⁡min⁡y^ 1N∑i=1Nℓ(yi,y^(xi)).\hat{y}_{\mathcal{D}} \coloneqq \underset{\hat{y}}{\arg\min} \ L_{\mathcal{D}}(\hat{y}) = \underset{\hat{y}}{\arg\min} \ \frac{1}{N} \sum_{i=1}^{N} \ell(\mathsf{y}_i, \hat{y}(\mathbf{x}_i)).y^​D​:=y^​argmin​ LD​(y^​)=y^​argmin​ N1​i=1∑N​ℓ(yi​,y^​(xi​)).

But, how do we learn an approximation of pD(y∣x)p_{\mathcal{D}}(y \mid \mathbf{x})pD​(y∣x) of p(y∣x)p(\mathsf{y} \mid \mathbf{x})p(y∣x) based on D\mathcal{D}D? If we can select a family of parametric probabilistic models (i.e., hypothesis classes), we can learn the model parameters to fit D\mathcal{D}D.

Linear Regression: Problem Setting

Definition 3 (Linear Regression Problem Setting)

In linear regression, we are interested in values of a fucntion y(x):Rd→Ry(\mathbf{x}): \mathbb{R}^d \to \mathbb{R}y(x):Rd→R, where x=(x1,…,xd)T\mathbf{x} = (x_1, \ldots, x_d)^Tx=(x1​,…,xd​)T. We have some observations of this mapping, precisely D={(xi,yi)}i=1N\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^{N}D={(xi​,yi​)}i=1N​.

Thus, the goal is to predict yyy for a new input x\mathbf{x}x, i.e., learn an accurate prediction function y^(x)\hat{y}(\mathbf{x})y^​(x) (also called regression function) from D\mathcal{D}D. Further, in linear regression, we assume that the relationship between x\mathbf{x}x and yyy is linear,

t(x)=w0+w1x1+…+wdxd=μ(x,w)\begin{align*} t(\mathbf{x}) & = w_0 + w_1 x_1 + \ldots + w_d x_d \newline & = \mu(\mathbf{x}, \mathbf{w}) \end{align*}t(x)​=w0​+w1​x1​+…+wd​xd​=μ(x,w)​

Probabilistic Modeling

Intuition (Probabilistic Modeling)

We can (again) further extend our framework by expressing our uncertainty over the value of μ(x,w)\mu(\mathbf{x}, \mathbf{w})μ(x,w) as,

y(x,w)=wTϕ(x)+ϵ=μ(x,w)+ϵ,ϵ∼N(0,β−1).y(\mathbf{x}, \mathbf{w}) = \mathbf{w}^T \boldsymbol{\phi}(\mathbf{x}) + \epsilon = \mu(\mathbf{x}, \mathbf{w}) + \epsilon, \quad \epsilon \sim \mathcal{N}(0, \beta^{-1}).y(x,w)=wTϕ(x)+ϵ=μ(x,w)+ϵ,ϵ∼N(0,β−1).

Thus, given x\mathbf{x}x, yyy can be modeled by a probabilistic model,

y∣x=x∼N(μ(x,w),β−1),\mathsf{y} \mid \mathbf{x} = \mathbf{x} \sim \mathcal{N}(\mu(\mathbf{x}, \mathbf{w}), \beta^{-1}),y∣x=x∼N(μ(x,w),β−1),

or, equivalently,

p(y∣x,w,β)=p(y∣x,θ)=N(y∣μ(x,w),β−1),p(\mathsf{y} \mid \mathbf{x}, \mathbf{w}, \beta) = p(\mathsf{y} \mid \mathbf{x}, \boldsymbol{\theta}) = \mathcal{N}(\mathsf{y} \mid \mu(\mathbf{x}, \mathbf{w}), \beta^{-1}),p(y∣x,w,β)=p(y∣x,θ)=N(y∣μ(x,w),β−1),

where θ=(w,β)\boldsymbol{\theta} = (\mathbf{w}, \beta)θ=(w,β).

Discriminative VS. Generative Probabilistic Models

In machine learning, there is a concept of discriminative and generative models 1 2.

Definition 4 (Discriminative Models)

In discriminative models, our posterior (predictive) distribution p(y∣x)p(y \mid \mathbf{x})p(y∣x) is assumed to belong to a hypothesis class parameterized by θ\boldsymbol{\theta}θ, i.e., p(y∣x,θ)p(y \mid \mathbf{x}, \boldsymbol{\theta})p(y∣x,θ). We learn the parameters θ\boldsymbol{\theta}θ directly from the training data D\mathcal{D}D.

Thus, once the model is learned, we can compute,

y^D(x)=arg⁡min⁡y^ Ey∼pD(y∣x)[ℓ(y,y^)∣x].\hat{y}_{\mathcal{D}}(\mathbf{x}) = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{y} \sim p_{\mathcal{D}}(y \mid \mathbf{x})}[\ell(\mathsf{y}, \hat{y}) \mid \mathbf{x}].y^​D​(x)=y^​argmin​ Ey∼pD​(y∣x)​[ℓ(y,y^​)∣x].
Example 2 (Linear Regression)

We first learn θ=(w,β)\boldsymbol{\theta} = (\mathbf{w}, \beta)θ=(w,β) based on D\mathcal{D}D. Then, we can find the optimal prediction for a new input x\mathbf{x}x as (under a quadratic loss function),

y^D(x)=Ey∼p(y∣x,θD⋆)[y∣x]\hat{y}_{\mathcal{D}}(\mathbf{x}) = \mathbb{E}_{\mathsf{y} \sim p(y \mid \mathbf{x}, \boldsymbol{\theta}^{\star}_{\mathcal{D}})}[\mathsf{y} \mid \mathbf{x}]y^​D​(x)=Ey∼p(y∣x,θD⋆​)​[y∣x]

y^D(x)=μ(x,wD⋆)\hat{y}_{\mathcal{D}}(\mathbf{x}) = \mu(\mathbf{x}, \mathbf{w}^{\star}_{\mathcal{D}})y^​D​(x)=μ(x,wD⋆​)
Definition 5 (Generative Models)

In generative models, we see the models p(x,y)p(\mathbf{x}, y)p(x,y) as being part of a parametric family p(x,y∣θ)p(\mathbf{x}, y \mid \boldsymbol{\theta})p(x,y∣θ). This means we from these models, can find p(x)p(\mathbf{x})p(x).

This is the generative ascpet of these models. We have the capacity to generate a realization of x\mathbf{x}x by using the marginal p(x∣θ)p(\mathbf{x} \mid \boldsymbol{\theta})p(x∣θ). Thus, once the model is learned, we can obtain p(y∣x,θ)p(y \mid \mathbf{x}, \boldsymbol{\theta})p(y∣x,θ) via Bayes’ theorem and computing,

y^D(x)=arg⁡min⁡y^ Ey∼pD(y∣x)[ℓ(y,y^)∣x].\hat{y}_{\mathcal{D}}(\mathbf{x}) = \underset{\hat{y}}{\arg\min} \ \mathbb{E}_{\mathsf{y} \sim p_{\mathcal{D}}(y \mid \mathbf{x})}[\ell(\mathsf{y}, \hat{y}) \mid \mathbf{x}].y^​D​(x)=y^​argmin​ Ey∼pD​(y∣x)​[ℓ(y,y^​)∣x].
Note

By making stronger assumptions, it (might) lead to more (significant) biases. However, in this setting we also have the ability to deal with missing data or latent variables (semi-supervised learning).

Learning the Model Parameters

We have clearly defined our problem setting and the overall goal. However, we have not stated how we actually learn/estimate the model parameters. As we discussed in the last part, we have two main approaches, the frequentist and the Bayesian approach.

Training, Generalization Loss, Bias, and Estimation Error

As the number of data points increases, overfitting is avoided. When D\mathcal{D}D is big compared to the number of parameters in θ\boldsymbol{\theta}θ,

The bias (approximation error) is caused by our choice of the hypothesis class (model misspecification) and the estimation error (generalization gap) is caused byt the fact that NNN is not large enough.

Learning and Validation

So, we would like to choose a model (hyperparameters and parameters) such that the generalization error Lp(y^)=Ex,y∼p(x,y)[ℓ(y,y^(x))]L_p(\hat{y}) = \mathbb{E}_{\mathbf{x}, \mathsf{y} \sim p(\mathbf{x}, y)}[\ell(\mathsf{y}, \hat{y}(\mathbf{x}))]Lp​(y^​)=Ex,y∼p(x,y)​[ℓ(y,y^​(x))] is minimized. However, it depends on p(x,y)p(\mathbf{x}, y)p(x,y), which is unknown.

Intuition (Validation)

We can divide our available data into three sets.

  • Training set D\mathcal{D}D to fit the model parameters.

  • Validation set V\mathcal{V}V to choose hypothesis class via evaluation of approximation of the generalization error,

Lp(w)≈1NV∑i=1NVℓ(yi,μ(xi,w)),L_p(\mathbf{w}) \approx \frac{1}{N_{\mathcal{V}}} \sum_{i=1}^{N_{\mathcal{V}}} \ell(\mathsf{y}_i, \mu(\mathbf{x}_i, \mathbf{w})),Lp​(w)≈NV​1​i=1∑NV​​ℓ(yi​,μ(xi​,w)),

for the selected hypothesis class, retrain w\mathbf{w}w based on D∪V\mathcal{D} \cup \mathcal{V}D∪V.

  • Test set T\mathcal{T}T to produce the estimate generalization error obtained with the final model.

However, there is a pitfall with this approach, parts of the available data are not used for training! Validation is suitable when we have plently of data. However, when data is scarce, we can use cross-validation (CV) techniques.

Algorithm (kkk-Fold Cross-Validation)

  1. Randomly partition data points into kkk partitions (folds)

  2. For each κ∈{1,…,k}\kappa \in \{1, \ldots, k\}κ∈{1,…,k}, train model over all other k−1k -1k−1 partitions.

  3. Compute generalization error on the κ\kappaκ-th partition,

  4. Generalization error estimated as average over all partitions.

  5. Choose hypothesis class that minimizes the estimate of the generalization error in Step 4.

Maximum A Posteriori Learning

Intuition (Maximum A Posteriori Learning)

In the Bayesian approach, we instead treat the parameters w\mathbf{w}w as unknown random variables (instead of unknown but deterministic quantities). MAP enables a finer control of the bias and estimation error.

The key idea is to leverage prior information available on the behavior of parameters in the absence, or presence, of overfitting.

Note

A large value of ∥w∥1\Vert \mathbf{w} \Vert_1∥w∥1​ a manifestation of overfitting ⇒\Rightarrow⇒ introduce a prior on w\mathbf{w}w that gives lower probability to larger values,

Summary (Bayesian Linear Regression)

In this part, we have seen that the frequentist approach, aims at identifying a specific value for θ\boldsymbol{\theta}θ of a probabilistic model to derive a predictor,

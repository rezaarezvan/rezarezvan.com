Introduction

In this part we will introduce linear models for classification including logistic regression, linear discriminant analysis (LDA), and quadratic discriminant analysis (QDA).

The Classification Problem

Intuition (Classification Problem)

In classification, the goal is, given a training set D≔{(x1,y1=,…,(xN,yN)}\mathcal{D} \coloneqq \{(\mathbf{x}_1, y_1=, \ldots, (\mathbf{x}_N , y_N)\}D:={(x1​,y1​=,…,(xN​,yN​)}, with yi∈{C1,…,Ck}y_i \in \{C_1, \ldots, C_k\}yi​∈{C1​,…,Ck​}, assign (classify) a new example x\mathbf{x}x to one of the classes CiC_iCi​. Usually, for convenience yi∈{1,…,K}y_i \in \{1, \ldots, K\}yi​∈{1,…,K}.

More mathematically, we can think of it as dividing the input space into decision regions Ri,i∈[K]\mathcal{R}_i, i \in [K]Ri​,i∈[K] (points in Ri\mathcal{R}_iRi​ are assigned to class CiC_iCi​). Thus, for linear models, the decision boundaries (surfaces) are linear functions of x\mathbf{x}x (i.e., (d−1)(d - 1)(d−1)-dimensional hyperplanes).

Intuition (Modeling Approaches)

There are three main approaches to modeling classification problems.

Discriminative Deterministic Models: Model directly the deterministic mapping between x\mathbf{x}x and yyy via a parametrized function μ(x,w)\mu(\mathbf{x}, \mathbf{w})μ(x,w) (discriminant function).

However, a more powerful approach is to model p(Ck∣x)p(C_k \mid \mathbf{x})p(Ck​∣x) in an inference stage, then use this distribution to make optimal decisions.

Discriminative Probabilistic Models: Model p(Ck∣x)p(C_k \mid \mathbf{x})p(Ck​∣x) via a parametrized model.

Generative Probabilistic Models: Model p(x,y)p(\mathbf{x}, y)p(x,y) specifying p(y)p(y)p(y) (p(Ck)p(C_k)p(Ck​)) and p(x∣y)p(\mathbf{x} \mid y)p(x∣y) (p(x∣Ck)p(\mathbf{x} \mid C_k)p(x∣Ck​)). Then,

p(Ck∣x)=p(x∣Ck)p(Ck)p(x)p(C_k \mid \mathbf{x}) = \frac{p(\mathbf{x} \mid C_k) p(C_k)}{p(\mathbf{x})}p(Ck​∣x)=p(x)p(x∣Ck​)p(Ck​)​
Intuition (Generalized Linear Models and Activation Functions)

Recall, in the linear regression setting, μ(x,w)\mu(\mathbf{x}, \mathbf{w})μ(x,w) is a linear function of w\mathbf{w}w, where in the simplest case we had μ(x,w)=wTx+w0\mu(\mathbf{x}, \mathbf{w}) = \mathbf{w}^T \mathbf{x} + w_0μ(x,w)=wTx+w0​. However, in classification, we need to ensure that the output is a discrete range or the posterior probabilities (i.e., [0,1][0, 1][0,1] and sum to one). Thus, generalized linear models are models where μ(x,w)=a(wTx)\mu(\mathbf{x}, \mathbf{w}) = a(\mathbf{w}^T \mathbf{x})μ(x,w)=a(wTx), where a(⋅)a(\cdot)a(⋅) is an activation function that maps the output of the linear function to the desired range.

For example, in binary classification, using a discriminative deterministic model, the simplest form would be,

In the more general multi-class setting, a single KKK-class discriminant with KKK linear functions,

One can interpret μk(x)\mu_k(\mathbf{x})μk​(x) as p(y=k∣x)p(y = k \mid \mathbf{x})p(y=k∣x). Further, the decision boundary between CkC_kCk​ and CjC_jCj​ is given by the hyperplane,

Linear (Probabilistic) Models for Classification

As we have discussed, discriminative probabilistic models aim at modeling p(Ck∣x)p(C_k \mid \mathbf{x})p(Ck​∣x) directly. Which are potentially more powerful than deterministic models since it allows us to model sources of uncertainty in the label assignment to input variables.

We will consider linear decision boundaries (in the input space) but also applying a nonlinear transformation ϕ=(ϕ0,…,ϕM)\boldsymbol{\phi} = (\phi_0, \ldots, \phi_M)ϕ=(ϕ0​,…,ϕM​) to x\mathbf{x}x, then work with ϕ(x)\boldsymbol{\phi}(\mathbf{x})ϕ(x) instead of x\mathbf{x}x. For clarity, decision boundaries linear in the feature space ϕ(x)\boldsymbol{\phi}(\mathbf{x})ϕ(x) are also called linear models, but they are nonlinear in the input space x\mathbf{x}x.

Thus, inspired by the linear regression model, we consider p(y=C1∣x)p(\mathsf{y} = C_1 \mid \mathbf{x})p(y=C1​∣x) of the form,

p(y=C1∣x,w)=f(wTx),0≤f(⋅)≤1p(\mathsf{y} = C_1 \mid \mathbf{x}, \mathbf{w}) = f(\mathbf{w}^T \mathbf{x}), \quad 0 \leq f(\cdot) \leq 1p(y=C1​∣x,w)=f(wTx),0≤f(⋅)≤1

A popular choice is the logit (or logistic sigmoid) function,

Recall (Maximum A Posteriori Learning)

ML can lead to overfitting when too many parameters are used compared to training examples. That is, a large value of ∥w∥\Vert \mathbf{w} \Vert∥w∥ a manifestation of overfitting. By introducing a prior on w\mathbf{w}w that gives lower probability to larger values, e.g., w∼N(0,α−1I)\mathbf{w} \sim \mathcal{N}(0, \alpha^{-1} \mathbf{I})w∼N(0,α−1I), we can use MAP learning to mitigate overfitting. Further, recall that the MAP criterion is defined as,

Generative Probabilistic Models for Classification

Intuition (Generative Probabilistic Models for Classification)

As we have discussed, making more assumptions about the data by considering distribution of covariates x\mathbf{x}x, we may suffer from bias when our model is incorrectly selected. The capability to capture properties of x\mathbf{x}x can improve learning if p(x∣y)p(\mathbf{x} \mid y)p(x∣y) has significant structure.

Since different models for p(x∣Ci)p(\mathbf{x} \mid C_i)p(x∣Ci​) lead to different generative models, generative models are typically defined by assuming,

x∣y=y∼exponential(ηy)\mathbf{x} \mid \mathsf{y} = y \sim \mathrm{exponential}(\eta_y)x∣y=y∼exponential(ηy​)

p(x∣η)=1Z(η)h(x)exp⁡(ηTu(x))p(\mathbf{x} \mid \boldsymbol{\eta}) = \frac{1}{Z(\boldsymbol{\eta})} h(\mathbf{x}) \exp(\boldsymbol{\eta}^T \mathbf{u}(\mathbf{x}))p(x∣η)=Z(η)1​h(x)exp(ηTu(x))

For example, Gaussian Discriminant Analysis (GDA), the class-conditional density is modeled by a multivariate Gaussian.

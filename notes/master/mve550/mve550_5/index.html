Introduction

In this part, we will introduce branching processes, their properties, some useful theorems, and applications. Then, we will introduce Markov Chain Monte Carlo (MCMC) and some algorithms related to it.

Branching Processes

Intuition: Branching Processes

Many real-world phenomena can be described as developing with a tree-like structure, for example,

  • Growth of cells.

  • Spread of viruses or other pathogens in a population.

  • Nuclear chain reactions.

  • Spread of funny cat videos on the internet.

  • Spread of a surname over generations. 1This was the original motivation for studying branching processes by Francis Galton and Henry William Watson in the 19th century.

The process with which one node gives rise to “children” can be described as random. We will assume the probabilistic properties of this process is the same for all nodes.

Further, we will assume all nodes are organized into generatiomns, we are only concerned with the size of each generation.

Lastly, the important questions we are interested in are; How large are the generations? How much does the size vary? Will the process die out eventually (i.e., reach a generation with size 0)?

Definition: Galton-Watson Branching Process

A (Galton-Watson) branching process is a discrete Markov chain Z0,Z1,…,Zn,…Z_0, Z_1, \ldots, Z_n, \ldotsZ0​,Z1​,…,Zn​,… where,

  • The state space is the non-negative integers.

  • Z0=1Z_0 = 1Z0​=1 (i.e., the process starts with one individual).

  • ZnZ_nZn​ is the sum X1+X2+…+XZn−1X_1 + X_2 + \ldots + X_{Z_{n - 1}}X1​+X2​+…+XZn−1​​, where XiX_iXi​ are independent random non-negative integers all with the same offspring distribution,

Connecting each of the ZnZ_nZn​ individuals in generation nnn with their offspring in generation n+1n + 1n+1 we get a tree illustrating the branching process.

The offspring distribution is described by the probability vector a=(a0,a1,…)a = (a_0, a_1, \ldots)a=(a0​,a1​,…) where aj=P(Xi=j)a_j = P(X_i = j)aj​=P(Xi​=j).

We will mainly focus on the interesting cases where we assume a0>0a_0 > 0a0​>0 (i.e., there is a positive probability of having no offspring) and a0+a1<1a_0 + a_1 < 1a0​+a1​<1 (i.e., there is a positive probability of having more than one offspring).

Markov Chain Monte Carlo (MCMC)

Intuition: Markov Chain Monte Carlo (MCMC)

So far we have used Markov chains in the following way

  1. Define a Markov chain

  2. Find its limiting distribution, if it exists

Now, we want to,

  1. Start with a desired distribution

Construct a Markov chain having this limiting distribution

In many contexts we want to compute the expecation E[r(X)]\mathbb{E}[r(X)]E[r(X)] for some random variable XXX and some function rrr.

One way to approximate this is to generate samples x1,…,xNx_1, \ldots, x_Nx1​,…,xN​ from XXX and use,

E[r(X)]=lim⁡N→∞1N∑k=1Nr(xk)≈1N∑k=1Nr(xk)\mathbb{E}[r(X)] = \lim_{N \to \infty} \frac{1}{N} \sum_{k = 1}^{N} r(x_k) \approx \frac{1}{N} \sum_{k = 1}^{N} r(x_k)E[r(X)]=N→∞lim​N1​k=1∑N​r(xk​)≈N1​k=1∑N​r(xk​)

Thus, the idea is simulating from a Markov chain having XXX as its limiting distribution generates an approximate sample. Average over it as above.

Intuition: Is an approximate sample good enough?

We know this works in classical statistics.

Strong law of large numbers of samples: If Y1,Y2,…,YmY_1, Y_2, \ldots, Y_mY1​,Y2​,…,Ym​ and YYY are i.i.d random variables from a distribution with finite mean, and if E[r(Y)]\mathbb{E}[r(Y)]E[r(Y)] exists, then, with probability 1,

lim⁡m→∞r(Y1)+r(Y2)+…+r(Ym)m=E[r(Y)]\lim_{m \to \infty} \frac{r(Y_1) + r(Y_2) + \ldots + r(Y_m)}{m} = \mathbb{E}[r(Y)]m→∞lim​mr(Y1​)+r(Y2​)+…+r(Ym​)​=E[r(Y)]

There is a corresponding result for Markov chains.

Strong law of large numbers for Markov chains: If X0,X1,…X_0, X_1, \ldotsX0​,X1​,… is an ergodic Markov chain with stationary distribution π\piπ, and if EX∼π[r(X)]\mathbb{E}_{X \sim \pi}[r(X)]EX∼π​[r(X)] exists, then, with probability 1,

lim⁡m→∞r(X0)+r(X1)+…+r(Xm)m=EX∼π[r(X)]\lim_{m \to \infty} \frac{r(X_0) + r(X_1) + \ldots + r(X_m)}{m} = \mathbb{E}_{X \sim \pi}[r(X)]m→∞lim​mr(X0​)+r(X1​)+…+r(Xm​)​=EX∼π​[r(X)]

where XXX has distribution π\piπ.

Note

When using this theorem in practice, one might improve accuracy by throwing away the first sequence X1,…,XsX_1, \ldots, X_sX1​,…,Xs​ for s<ms < ms<m before computing the average. The first sequence is then called the burn-in.

The Metropolis-Hastings Algorithm, Gibbs Sampling, and The Ising Model

Putting together an algorithm updating different coordinates in different steps may create an ergodic Markov chain.

This is called Gibbs sampling. Sometimes the conditional distributions are easy to derive. Then this is an easy-to-use version of Metropolis-Hastings.

Definition: The Ising Model

Uses a grid of vertices; We will assume an n×nn \times nn×n grid. Two vertices vvv and www are neighbours, denoted v∼wv \sim wv∼w, if they are next to each other in the grid.

Each vertex vvv can have value +1+1+1 or −1-1−1 (called its “spin”); We denote this by σv=1\sigma_{v} = 1σv​=1 or σv=−1\sigma_{v} = -1σv​=−1.

A configuration σ\sigmaσ consists of a choice of +1+1+1 or −1-1−1 for each vertex. Thus, the set Ω\OmegaΩ of possible configurations has 2(n2)2^(n^2)2(n2) elements. We define the energy of a configuration as E(σ)=−∑v∼wσvσwE(\sigma) = - \sum_{v \sim w} \sigma_v \sigma_wE(σ)=−∑v∼w​σv​σw​.

The Gibbs distribution is the probability mass function on Ω\OmegaΩ defined by,

π(σ)∝exp⁡(−βE(σ))\pi(\sigma) \propto \exp(-\beta E(\sigma))π(σ)∝exp(−βE(σ))

where β\betaβ is a parameter of the model; 1β\frac{1}{\beta}β1​ is called the temperature.

It turns out that when the temperature is high, samples from the model will show a chaotic pattern of spins, but when the temperature sinks below the phase transition value, samples will show chunks of neighbouring vertices with the same spin, i.e., the system will be “magnetized”.

Perfect Sampling

So far, we have seen that MCMC means,

  • We want to find the expected value of some expression under some distribution.
  • The distribution is difficult to sample from, so instead we simulate from a Markov chain having the given distribution as its limiting distribution.
  • Using the strong law of large numbers for Markov chains, we can approximate the expected value as an average over expressions computed on your simulated values.

When we sample directly from a distribution, the rate of convergence of the averages is governed by the central limit theorem. When using a Markov chain, it is generally very difficult to find the rate of convergence.

There are some exceptions to this, we will now look at Perfect Sampling, where you prove that you have reached the limiting distribution exactly.

Intuition: Perfect Sampling

Given an ergodic Markov chain with finite state space of size kkk and limiting distribution π\piπ. The idea is that, given nnn prove that XnX_nXn​ actually has reached the limit distribution π\piπ.

If we can rpove that the distribution at XnX_nXn​ is independent of the starting state X0X_0X0​, then we might be able to conclude that XnX_nXn​ has the limiting distribution π\piπ.

Construct kkk Markov chains that are dependent (“coupled”) but which are marginally Markov chains as above. If they start at the kkk possible value at X0X_0X0​ but have identical values at XnX_nXn​, then we are done.

Definition: Monotonicity

One smart question is, do we need to keep track of all kkk chains? We define a partial ordering on as et as a relation x≤yx \leq yx≤y between some pairs xxx and yyy in the set, such that,

  • If x≤yx \leq yx≤y and y≤xy \leq xy≤x then x=yx = yx=y.

  • If x≤yx \leq yx≤y and y≤zy \leq zy≤z then x≤zx \leq zx≤z.

We will need that our partial ordering has a minimal element (an mmm such that m≤xm \leq xm≤x for all xxx) and a maximal element (an MMM such that x≤Mx \leq Mx≤M for all xxx).

If we have a partial ordering on the state space of the Markov chain and if x≤yx \leq yx≤y implies g(x,U)≤g(y,U)g(x, U) \leq g(y, U)g(x,U)≤g(y,U), then ggg is monotone.

We can then prove that we only need to keep track of the chain starting at mmm and the chain starting at MMM.

Proof

If the chains starting at mmm and MMM coalesce at time 0, then for any other chain starting at xxx, we have m≤x≤Mm \leq x \leq Mm≤x≤M. By monotonicity, we get that the chain starting at mmm is less than or equal to the chain starting at xxx at all times, and the chain starting at MMM is greater than or equal to the chain starting at xxx at all times. Thus, if the chains starting at mmm and MMM are equal at time 0, the chain starting at xxx must also be equal to them at time 0.

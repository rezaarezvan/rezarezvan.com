Review

Let’s review over what we covered last time.

Convex Set

Recall that a line segment between x(1)\mathbf{x}^{(1)}x(1) and x(2)\mathbf{x}^{(2)}x(2) all points,

x=αx(1)+(1−α)x(2)\mathbf{x} = \alpha \mathbf{x}^{(1)} + (1 - \alpha) \mathbf{x}^{(2)}x=αx(1)+(1−α)x(2)

So, a convex set, contains line segment between any two points in the set,

x(1),x(2)∈χ,0≤α≤1⇒αx(1)+(1−α)x(2)∈χ.\mathbf{x}^{(1)}, \mathbf{x}^{(2)} \in \chi, 0 \leq \alpha \leq 1 \Rightarrow \alpha \mathbf{x}^{(1)} + (1 - \alpha) \mathbf{x}^{(2)} \in \chi.x(1),x(2)∈χ,0≤α≤1⇒αx(1)+(1−α)x(2)∈χ.

f:RN→Rf : \mathbb{R}^N \rightarrow \mathbb{R}f:RN→R is convex if dom(f)dom(f)dom(f) is a convex set and,

f(αx(1)+(1−α)x(2))≤αf(x(1))+(1−α)f(x(2))f(\alpha \mathbf{x}^{(1)} + (1 - \alpha) \mathbf{x}^{(2)}) \leq \alpha f(\mathbf{x}^{(1)}) + (1 - \alpha) f(\mathbf{x}^{(2)})f(αx(1)+(1−α)x(2))≤αf(x(1))+(1−α)f(x(2))

Also, remember that fff is concave if −f-f−f is convex.

First Order Condition

fff is differentiable if the gradient,

∇f(x)=[∂f(x)∂x1,∂f(x)∂x2,…,∂f(x)∂xN]T\nabla f(\mathbf{x}) = \begin{bmatrix} \frac{\partial f(\mathbf{x})}{\partial x_1}, \frac{\partial f(\mathbf{x})}{\partial x_2}, \ldots, \frac{\partial f(\mathbf{x})}{\partial x_N} \end{bmatrix}^T∇f(x)=[∂x1​∂f(x)​,∂x2​∂f(x)​,…,∂xN​∂f(x)​​]T

exists at each x∈dom(f)\mathbf{x} \in dom(f)x∈dom(f).

The first-order condition is that, differentiable fff with convex domain is convex if and only if,

f(y)≥f(x)+∇f(x)T(y−x)∀x,y∈dom(f).f(\mathbf{y}) \geq f(\mathbf{x}) + \nabla f(\mathbf{x})^T (\mathbf{y} - \mathbf{x}) \quad \forall \mathbf{x}, \mathbf{y} \in dom(f).f(y)≥f(x)+∇f(x)T(y−x)∀x,y∈dom(f).

First-order approximation of fff is global underestimator.

Second-Order Condition

fff is twice differentiable if the Hessian ∇2f(x)\nabla^2 f(\mathbf{x})∇2f(x) exists at each x∈SN\mathbf{x} \in \mathbb{S}^Nx∈SN,

∇2f(x)ij=∂2f∂xi∂xj.\nabla^2 f(\mathbf{x})_{ij} = \frac{\partial^2 f}{\partial x_i \partial x_j}.∇2f(x)ij​=∂xi​∂xj​∂2f​.

exists at each x∈dom(f)\mathbf{x} \in dom(f)x∈dom(f).

The second-order condition is that, twice differentiable fff with convex domain is convex if and only if,

∇2f(x)⪰0\nabla^2 f(\mathbf{x}) \succeq 0∇2f(x)⪰0

This means that the Hessian matrix is positive semidefinite,

A⪰0⇔∀x,xTAx≥0.\mathbf{A} \succeq 0 \Leftrightarrow \forall \mathbf{x}, \mathbf{x}^T \mathbf{A} \mathbf{x} \geq 0.A⪰0⇔∀x,xTAx≥0.

Convex Optimization Problems

To understand what a convex optimization problem is, we need to understand optimization problems in standard form.

Optimization Problem in Standard Form

minimizexf0(x)subject tofi(x)≤0,i=1,…,rhi(x)=0,i=1,…,s\begin{aligned} \underset{\mathbf{x}}{\text{minimize}} & \quad f_0(\mathbf{x}) \newline \text{subject to} & \quad f_i(\mathbf{x}) \leq 0, \quad i = 1, \ldots, r \newline & \quad h_i(\mathbf{x}) = 0, \quad i = 1, \ldots, s \end{aligned}xminimize​subject to​f0​(x)fi​(x)≤0,i=1,…,rhi​(x)=0,i=1,…,s​

where,

  • x∈RN\mathbf{x} \in \mathbb{R}^Nx∈RN is the optimization variable,
  • f0:RN→Rf_0 : \mathbb{R}^N \rightarrow \mathbb{R}f0​:RN→R is the objective function or cost function,
  • fi:RN→Rf_i : \mathbb{R}^N \rightarrow \mathbb{R}fi​:RN→R, i=1,…,ri = 1, \ldots, ri=1,…,r, are the inequality constraint functions,
  • hi:RN→Rh_i : \mathbb{R}^N \rightarrow \mathbb{R}hi​:RN→R, i=1,…,si = 1, \ldots, si=1,…,s, are the equality constraint functions.

The optimal value of the optimization problem is,

p⋆=min⁡{f0(x∣fi(x)≤0,i=1,…,r,hi(x)=0,i=1,…,s)}p^{\star} = \min \{f_0(\mathbf{x} | f_i(\mathbf{x}) \leq 0, i = 1, \ldots, r, h_i(\mathbf{x}) = 0, i = 1, \ldots, s)\}p⋆=min{f0​(x∣fi​(x)≤0,i=1,…,r,hi​(x)=0,i=1,…,s)}

Note,

  • p⋆=∞p^{\star} = \inftyp⋆=∞ if the problem is infeasible (no x\mathbf{x}x satisfies the constraints).
  • p⋆=−∞p^{\star} = -\inftyp⋆=−∞ if the problem is unbounded (the objective function can be made arbitrarily small).

Local and Global Optima in Convex Optimization

Any locally optimal point of is (globally) optimal.

Proof. Suppose x\mathbf{x}x is locally optimal and y\mathbf{y}y is optimal with f0(y)<f0(x)f_0(\mathbf{y}) < f_0(\mathbf{x})f0​(y)<f0​(x). Locally optimal x\mathbf{x}x means there is a R>0R > 0R>0 such that,

z feasible, ∥z−x∥2≤R⇒f0(z)≥f0(x).\mathbf{z} \text{ feasible, } \Vert \mathbf{z} - \mathbf{x} \Vert_2 \leq R \Rightarrow f_0(\mathbf{z}) \geq f_0(\mathbf{x}).z feasible, ∥z−x∥2​≤R⇒f0​(z)≥f0​(x).

Consider z=αy+(1−α)x\mathbf{z} = \alpha \mathbf{y} + (1 - \alpha) \mathbf{x}z=αy+(1−α)x, with α=R2∥y−x∥2\alpha = \frac{R}{2 \Vert \mathbf{y} - \mathbf{x} \Vert_2}α=2∥y−x∥2​R​. Thus, ∥y−x∥2>R\Vert \mathbf{y} - \mathbf{x} \Vert_2 > R∥y−x∥2​>R, so 0<α<120 < \alpha < \frac{1}{2}0<α<21​.

z\mathbf{z}z is a convex combination of two feasible points, hence feasible. ∥z−x∥2=R2\Vert \mathbf{z} - \mathbf{x} \Vert_2 = \frac{R}{2}∥z−x∥2​=2R​, and,

Duality

To understand duality, we need to understand the Lagrangian.

Lagrangian

Given the optimization problem in standard form (not necessarily convex),

minxf0(x)subject tofi(x)≤0,i=1,…,rhi(x)=0,i=1,…,s\begin{aligned} \underset{\mathbf{x}}{\text{min}} & \quad f_0(\mathbf{x}) \newline \text{subject to} & \quad f_i(\mathbf{x}) \leq 0, \quad i = 1, \ldots, r \newline & \quad h_i(\mathbf{x}) = 0, \quad i = 1, \ldots, s \end{aligned}xmin​subject to​f0​(x)fi​(x)≤0,i=1,…,rhi​(x)=0,i=1,…,s​

We have the variable x∈RN\mathbf{x} \in \mathbb{R}^Nx∈RN, domain X\ChiX and the optimal value p⋆p^{\star}p⋆.

Lagrangian L:RN×Rr×Rs→R\mathbf{L} : \mathbb{R}^N \times \mathbb{R}^r \times \mathbb{R}^s \rightarrow \mathbb{R}L:RN×Rr×Rs→R, with dom(L)=X×Rr×Rsdom(L) = \Chi \times \mathbb{R}^r \times \mathbb{R}^sdom(L)=X×Rr×Rs,

L(x,λ,ν)=f0(x)+∑i=1rλifi(x)+∑i=1sνihi(x).\mathbf{L}(\mathbf{x}, \mathbf{\lambda}, \mathbf{\nu}) = f_0(\mathbf{x}) + \sum_{i=1}^r \lambda_i f_i(\mathbf{x}) + \sum_{i=1}^s \nu_i h_i(\mathbf{x}).L(x,λ,ν)=f0​(x)+i=1∑r​λi​fi​(x)+i=1∑s​νi​hi​(x).

This can be interpreted as the weighted sum of the objective and the constraints functions.

λi\lambda_iλi​ is Lagrange multiplier associated with fi(x)≤0f_i(\mathbf{x}) \leq 0fi​(x)≤0. νi\nu_iνi​ is Lagrange multiplier associated with hi(x)=0h_i(\mathbf{x}) = 0hi​(x)=0.

The Dual Problem

The Lagrange dual problem is,

Find the best lower bound p⋆p^{\star}p⋆, obtained from Lagrange dual problem.

This is a convex optimization problem, the optimal value is denoted d⋆d^{\star}d⋆.

  • If we have weak duality: d⋆≤p⋆d^{\star} \leq p^{\star}d⋆≤p⋆.

    • Always holds (for convex problems and non-convex problems).
    • Can be used to find non-trivial lower bounds for difficult problems.

  • If we have strong duality: d⋆=p⋆d^{\star} = p^{\star}d⋆=p⋆.

    • Does not hold in general.
    • (Usually) holds for convex problems, e.g, SVM.

Support Vector Machine (SVM)

Recall from last time, we assumed the data is linearly separable, what if the data is separable, but not linearly separable?

A smart idea is to transform the input space. Map x∈RN\mathbf{x} \in \mathbb{R}^Nx∈RN to a new high-dimensional space z∈RL\mathbf{z} \in \mathbb{R}^Lz∈RL, z=ϕ(x)z = \phi(\mathbf{x})z=ϕ(x), where ϕ\phiϕ is the transform function.

Then, we can learn the linear classifier in this new space.

Kernel Function

Dual SVM with transformed input,

Completely written in terms of inner product ϕ(x(i))Tϕ(x(j))\phi(\mathbf{x}^{(i)})^T \phi(\mathbf{x}^{(j)})ϕ(x(i))Tϕ(x(j)).

Rather than explicitly calculate the high-dimensional ϕ(x)\phi(\mathbf{x})ϕ(x), we only need to calculate the inner product between the two vectors.

Let the kernel function K(x(i),x(j))=ϕ(x(i))Tϕ(x(j))\mathcal{K}(\mathbf{x}^{(i)}, \mathbf{x}^{(j)}) = \phi(\mathbf{x}^{(i)})^T \phi(\mathbf{x}^{(j)})K(x(i),x(j))=ϕ(x(i))Tϕ(x(j)).

This is much less expensive to compute than explicitly calculating the high-dimensional feature vector and the inner product.

Kernel Trick

Replacing the inner product with a kernel function in optimization is called the kernel trick.

Turns linear classifier into a non-linear classifier. The shape of the decision boundary is determined by the kernel function.

RBF Kernel

The Radial Basis Function (RBF) kernel is a popular kernel function,

K(x,x′)=e−γ∥x−x′∥2\mathcal{K}(\mathbf{x}, \mathbf{x}^{\prime}) = e^{-\gamma \Vert \mathbf{x} - \mathbf{x}^{\prime} \Vert^2}K(x,x′)=e−γ∥x−x′∥2

Which resembles a Gaussian.

Gamma γ>0\gamma > 0γ>0 is the inverse bandwidth parameter of the kernel. It controls the smoothness of the function.

Small γ→\gamma \rightarrowγ→ wide Gaussian →\rightarrow→ smooth function. Large γ→\gamma \rightarrowγ→ thin Gaussian →\rightarrow→ wiggly function.

Corresponds to feature transform function of infinite dimension.

Kernel SVM: Summary

  • Kernel Classifier.
    • Kernel function defines the shape of the non-linear decision boundary.
      • Implicitly transforms input feature into high-dimensional space.
      • Uses linear classifier in high-dimensional space.
      • The decision boundary is non-linear in the original space.
  • Training.
    • Maximize the margin of the training data.
      • I.e, maximizes the separation between the points and the decision boundary.
    • Use cross-validation to pick the hyperparameters and the kernel hyperparameters.
  • Advantages.
    • Non-linear decision boundary for more complex classification problems.
    • Some intuition from the type of kernel function used.
  • Disadvantages.
    • Sensitive to the kernel function used.
    • Sensitive to the CCC and kernel hyperparameters.
    • Computationally expensive to do cross-validation.
    • Need to calculate the kernel matrix
      • M2M^2M2 terms where MMM is the size of the training set.
      • For large MMM, uses large amount of computation and memory.

Sequential Minimal Optimization (SMO)

SMO is an algorithm for training SVMs. Let’s take a look.

Solving SVM: Coordinate Descent

Consider the unconstrained optimization problem,

min⁡θℓ(θ1,θ2,…,θN)\underset{\mathbf{\theta}}{\min} \ell(\theta_1, \theta_2, \ldots, \theta_N)θmin​ℓ(θ1​,θ2​,…,θN​)

The coordinate descent method first selects an initial point θ(0)∈RN\mathbf{\theta}^{(0)} \in \mathbb{R}^Nθ(0)∈RN and repeats:

  • Choose an index iii from 1 to NNN.
  • θi(t+1)=arg⁡min⁡θ^iℓ(θ1(t),…,θ^i,…,θN(t))\theta_i^{(t+1)} = \arg \min_{\hat{\theta}_i} \ell(\theta_1^{(t)}, \ldots, \hat{\theta}_i, \ldots, \theta_N^{(t)})θi(t+1)​=argminθ^i​​ℓ(θ1(t)​,…,θ^i​,…,θN(t)​)

At each iteration, the algorithm determines a coordinate, and minimizes over the corresponding coordinate direction while fixing all other coordinates.

Coordinate descent is applicable in both differentiable and non-differentiable optimization contexts.

Solving SVM: Sequential Minimal Optimization (SMO)

SMO is an algorithm for solving the quadratic programming problem that arises during the training of SVMs, invented by John C. Platt in 1998.

  • Sequential
    • Not parallel.
    • Optimize a set of two Lagrange multipliers.
  • Minimal
    • Optimize the smallest possible sub-problem at each step.
  • Optimization
    • Satisfy the constraints for the chosen pair of Lagrange multipliers.

SMO

Recall the dual form of kernelized SVM with soft margin,

max⁡α∑i=1Mαi−12∑i,j=1My(i)y(j)αiαjK(x(i),x(j))subject to∑i=1Mαiy(i)=00≤αi≤C,i=1,…,M\begin{aligned} \underset{\mathbf{\alpha}}{\max} & \quad \sum_{i=1}^M \alpha_i - \frac{1}{2} \sum_{i,j=1}^M y^{(i)} y^{(j)} \alpha_i \alpha_j \mathcal{K}(\mathbf{x}^{(i)}, \mathbf{x}^{(j)}) \newline \text{subject to} & \quad \sum_{i=1}^M \alpha_i y^{(i)} = 0 \newline & \quad 0 \leq \alpha_i \leq C, \quad i = 1, \ldots, M \end{aligned}αmax​subject to​i=1∑M​αi​−21​i,j=1∑M​y(i)y(j)αi​αj​K(x(i),x(j))i=1∑M​αi​y(i)=00≤αi​≤C,i=1,…,M​

SMO is derived by decomposing the problem to its extreme and optimizing a minimal subset of just two dual variables (i.e., two data points) at each iteration.

The sub-problem for two data points admits an analytical solution, eliminating the need to use an iterative quadratic programming optimizer as part of the algorithm.

The condition ∑i=1Mαiy(i)=0\sum_{i=1}^M \alpha_i y^{(i)} = 0∑i=1M​αi​y(i)=0 is enforced throughout the iterations, implying that the smallest number of multipliers that can be optimized at each step is two.

Whenever one multiplier is updated, at least one other multiplier needs to be adjusted in order to keep the condition true.

At each step SMO chooses two elements αi\alpha_iαi​ and αj\alpha_jαj​ to jointly optimize, finds the optimal values for those two parameters given that all the others are fixed.

The choice of the two points is determined by a heuristic, while the optimization of the two multipliers is performed analytically.

Despite needing more iterations to converge, each iteration uses so few operations that the algortihm exhibits an overall speed-up of some orders of magnitude.

    • Class yyy: from a set of possible classes, e.g., Y=−1,+1\mathcal{Y} = \newline{-1, +1\newline}Y=−1,+1.
    • Goal: given an observation x\mathbf{x}x, predict the class yyy.
  • K-Nearest Neighbors (KNN)
    • Type: Discriminative.
    • Classes: Multi-class.
    • Decision Function: Non-linear.
    • Training: No training needed.
  • Bayes’ Classifier
    • Type: Generative.
    • Classes: Multi-class.
    • Decision Function: (generally) Non-linear.
    • Training: Estimate class-conditional densities p(x∣y)p(\mathbf{x} | y)p(x∣y) by maximizing likelihood of data.
  • Logistic Regression
    • Type: Discriminative.
    • Classes: Binary.
    • Decision Function: Linear.
    • Training: Maximize likelihood of data in p(y∣x)p(y | \mathbf{x})p(y∣x).
  • Support Vector Machine (SVM)
    • Type: Discriminative.
    • Classes: Binary.
    • Decision Function: linear.
    • Training: Maximize the margin (distance between the decision surface and the closest point).
  • Kernel SVM
    • Type: Discriminative.
    • Classes: Binary.
    • Decision Function: Non-linear (kernel function).
    • Training: Maximize the margin.

Regulatization and Overfitting

Some models have terms to prevent overfitting the training data, this can improve generalization to new data.

There is a parameter to control the regularization effect, select this parameter using cross-validation on the training set.

Other Things

  • Multi-class classification.

    • Can use binary classifiers to do multi-class using 1-vs-rest formulation.

  • Feature Normalization.

    • Normalize each feature dimension so that some feature dimensions with larger ranges do not dominate the optimization process.

  • Data Imbalance

    • If more data in one class, then apply weights to each class to balance objectives.

  • Class Importance

    • Mistakes on some classes are more critical.
    • Reweight class to focus classifier on correctly predicting one class at the expense of others.

  • Applications

    • Web document classification, spam classification.
    • Face gender recognition, face detection, digit classification.

  • Features

    • Choice of features is important!
      • Using uniformative features may confuse the classifier.
      • Use domain knowledge to pick the best features to extract from the data.

Which Classifier is Best?

“No Free Lunch” Theorem (Wolpert and Macready)

If an algorithm performs well on a certain class of problems then it necessarily pays for that with degraded performance on the set of all remaining problems.

In other words, there is no best classifier for all tasks. The best classifier depends on the particular problem.

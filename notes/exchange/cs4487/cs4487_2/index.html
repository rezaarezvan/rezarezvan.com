The Bayes Optimal Classifier

Recall from the previous part, that the best classifier, in theory, is the Bayes Optimal Classifier. However, it is not useful in reality due to the difficulty of estimating p(x∣y=c)p(\mathbf{x} | y = c)p(x∣y=c) in practice.

Naive Bayes Classifier

The Naive Bayes classifier is a simplified version of the Bayes Optimal Classifier.

It has the Naive Bayes assumption, which assumes that all of the feature dimensions are statistically independant given the value of the class variable, for example,

p(x1,x2∣y)=p(x1∣y)p(x2∣y)p(x_1, x_2 | y) = p(x_1 | y) p(x_2 | y)p(x1​,x2​∣y)=p(x1​∣y)p(x2​∣y)

It also accumulates evidence from each feature dimension,

log⁡p(x1,x2∣y)=log⁡p(x1∣y)+log⁡p(x2∣y)\log p(x_1, x_2 | y) = \log p(x_1 | y) + \log p(x_2 | y)logp(x1​,x2​∣y)=logp(x1​∣y)+logp(x2​∣y)

This allows us to model each dimension of the observation with a simple univariate distribution.

Therefore, the general form for the classification is,

fNB(x)=arg⁡max⁡c∈{1,…,C}p(y=c)∏j=1Np(xj∣y=c)f_{NB}(\mathbf{x}) = \underset{c \in \{1, \ldots, C\}}{\arg\max} p(y = c) \prod_{j = 1}^{N} p(x_j | y = c)fNB​(x)=c∈{1,…,C}argmax​p(y=c)j=1∏N​p(xj​∣y=c)

Gaussian Classifier

If we treat each dimension as an independent Gaussian and fit the Gaussian parameters with MLE, we have a Gaussian classifier.

But what is the geometry of the decision boundary, let’s take a look at the 2D case.

Naive Bayes for Boolean Vectors

In the case we are working with boolean vectors, we can simplify the Naive Bayes classifier.

We still model each feature independently, but we can use the Bernoulli distribution to model the feature.

If there is a presence of a feature, we model it as,

p(xj=1∣y=c)=φj∣cp(x_j = 1 | y = c) = \varphi_{j | c}p(xj​=1∣y=c)=φj∣c​

and in the absence of a feature, we model it as,

p(xj=0∣y=c)=1−φj∣cp(x_j = 0 | y = c) = 1 - \varphi_{j | c}p(xj​=0∣y=c)=1−φj∣c​

Therefore, the MLE parameters are φj∣c=Mj∣c/Mc\varphi_{j | c} = M_{j | c} / M_cφj∣c​=Mj∣c​/Mc​, where, Mj∣cM_{j | c}Mj∣c​ is the number of training examples in class ccc that contain feature xjx_jxj​. McM_cMc​ is the number of training examples in class ccc.

We therefore get class-conditional distributions,

p(x1,…,xN∣y=c)=∏j=1Np(xj∣y=c)log⁡p(x1,…,xN∣y=c)=∑j=1Nlog⁡p(xj∣y=c)p(x_1, \ldots, x_N | y = c) = \prod_{j = 1}^{N} p(x_j | y = c) \newline \log p(x_1, \ldots, x_N | y = c) = \sum_{j = 1}^{N} \log p(x_j | y = c)p(x1​,…,xN​∣y=c)=j=1∏N​p(xj​∣y=c)logp(x1​,…,xN​∣y=c)=j=1∑N​logp(xj​∣y=c)

So, for an input example x\mathbf{x}x, the log-probabilities of features present given y=cy = cy=c adds.

Smoothing

Some features may not be present in any training examples for a given class, which means, Mj∣c=0M_{j | c} = 0Mj∣c​=0 and thus φj∣c=0\varphi_{j | c} = 0φj∣c​=0.

Which is problematic, since it may appear in a real-world example.

To solve this we need to smooth our MLE, by adding a smoothing parameter α\alphaα, which can be interpreted as a “pseudo-count”.

So, our new parameter is,

φj∣c=Mj∣c+αMc+Nα\varphi_{j | c} = \frac{M_{j | c} + \alpha}{M_c + N\alpha}φj∣c​=Mc​+NαMj∣c​+α​

If α=1\alpha = 1α=1, then we have Laplace Smoothing (also called additive smoothing).

In general, smoothing helps to prevent overfitting of the parameters.

What if xjx_jxj​ has a finite set of categories?

xj∈χj={1,…,∣χj∣}x_j \in \chi_j = \{1, \ldots, | \chi_j |\}xj​∈χj​={1,…,∣χj​∣}

Our MLE parameter, φx,j∣c=Mx,j∣c/Mj∣c\varphi_{x, j | c} = M_{x, j | c} / M_{j | c}φx,j∣c​=Mx,j∣c​/Mj∣c​, where, Mx,j∣cM_{x, j | c}Mx,j∣c​ is the number of training examples in class ccc that have feature xj=xx_j = xxj​=x. Mj∣cM_{j | c}Mj∣c​ is the number of training examples in class ccc.

Linear Discriminant Analysis (LDA)

LDA is a classification technique that dates back to the 1930s, with origins from Fisher.

It can be interpreted as a approximation to the Bayes optimal classifier, but for real-valued data.

Instead of a product of independent Gaussians, LDA assumes that p(x∣y=c)=N(x;μc,Σ)p(\mathbf{x} | y = c) = \mathcal{N}(\mathbf{x}; \mathbf{\mu}_c, \mathbf{\Sigma})p(x∣y=c)=N(x;μc​,Σ). A multivariate Gaussian with a shared covariance matrix Σ\mathbf{\Sigma}Σ.

Which equates to,

p(x∣y=c)=1∣(2π)NΣ∣12exp(−12(x−μc)TΣ−1(x−μc))p(\mathbf{x} | y = c) = \frac{1}{|(2\pi)^N \mathbf{\Sigma}}|^{\frac{1}{2}} exp\left(-\frac{1}{2} (\mathbf{x} - \mathbf{\mu}_c)^T \mathbf{\Sigma}^{-1} (\mathbf{x} - \mathbf{\mu}_c)\right)p(x∣y=c)=∣(2π)NΣ1​∣21​exp(−21​(x−μc​)TΣ−1(x−μc​))

The classification function is then,

fLDA(x)=arg⁡max⁡c∈{1,…,C}p(y=c)p(x∣y=c)f_{LDA}(\mathbf{x}) = \underset{c \in \{1, \ldots, C\}}{\arg\max} p(y = c) p(\mathbf{x} | y = c)fLDA​(x)=c∈{1,…,C}argmax​p(y=c)p(x∣y=c)

Training LDA

As with Naive Bayes, the LDA parameters are learned using MLE.

Our class probabilities are,

p(y=c)=1M∑i=1MI(y(i)=c)=McMp(y = c) = \frac{1}{M} \sum_{i = 1}^{M} \mathbb{I}(y^{(i)} = c) = \frac{M_c}{M}p(y=c)=M1​i=1∑M​I(y(i)=c)=MMc​​

Class means are,

μc=∑i=1MI(y(i)=c)x(i)∑i=1MI[y(i)=c]=∑i=1Mcx(i)Mc\mathbf{\mu_c} = \frac{\sum_{i = 1}^M \mathbb{I}(y^{(i)} = c) \mathbf{x}^{(i)}}{\sum_{i = 1}^M \mathbb{I}[y^{(i)} = c]} = \frac{\sum_{i = 1}^{M_c} \mathbf{x}^{(i)}}{M_c}μc​=∑i=1M​I[y(i)=c]∑i=1M​I(y(i)=c)x(i)​=Mc​∑i=1Mc​​x(i)​

where x(i)\mathbf{x}^{(i)}x(i) belongs to class ccc.

The shared covariance matrix is,

Σ=1M∑i=1M(x(i)−μy(i))(x(i)−μy(i))T\mathbf{\Sigma} = \frac{1}{M} \sum_{i = 1}^M \left(\mathbf{x}^{(i)} - \mathbf{\mu_{y^{(i)}}}\right) \left(\mathbf{x}^{(i)} - \mathbf{\mu_{y^{(i)}}}\right)^TΣ=M1​i=1∑M​(x(i)−μy(i)​)(x(i)−μy(i)​)T

Geometric Interpretation

Let’s take a look at the geometry of the decision boundry for LDAs.

The decision boundary is the set of points x\mathbf{x}x where,

log⁡p(y=1)−12log⁡∣(2π)2Σ∣−12(x−μ1)TΣ−1(x−μ1)−log⁡p(y=2)−12log⁡∣(2π)2Σ∣−12(x−μ2)TΣ−1(x−μ2)=0\log p(y = 1) - \frac{1}{2} \log |(2\pi)^2 \mathbf{\Sigma}| - \frac{1}{2}(\mathbf{x} - \mathbf{\mu}_1)^T \mathbf{\Sigma}^{-1} (\mathbf{x} - \mathbf{\mu}_1) \newline - \log p(y = 2) - \frac{1}{2} \log |(2\pi)^2 \mathbf{\Sigma}| - \frac{1}{2}(\mathbf{x} - \mathbf{\mu}_2)^T \mathbf{\Sigma}^{-1} (\mathbf{x} - \mathbf{\mu}_2) = 0logp(y=1)−21​log∣(2π)2Σ∣−21​(x−μ1​)TΣ−1(x−μ1​)−logp(y=2)−21​log∣(2π)2Σ∣−21​(x−μ2​)TΣ−1(x−μ2​)=0

This shows that the decision boundary is linear in x\mathbf{x}x.

Naive Bayes vs LDA

So, which is better?

Let’s break it down.

  • Storage
    • Naive Bayes: O(N)\mathcal{O}(N)O(N) parameters
    • LDA: O(N2)\mathcal{O}(N^2)O(N2) parameters
  • Speed The quadratic dependence on N makes LDA slower than NB during learning and classification.
  • Interpretability
    • Both models have good interpretability since the parameters of p(x∣y=c)p(\mathbf{x} | y = c)p(x∣y=c) corresponds to class conditional averages.
  • Data
    • LDA will generally need more data than NB since it needs to estimate O(N2)\mathcal{O}(N^2)O(N2) parameters in the shared covariance matrix.

Summary of Generative Models

So, what we have been looking at can also be called generative classification models.

They estimate a probability distribution of features from each class. Given a feature, predict the class with the largest posterior probability.

  • Advantages
    • Works with small amounts of data.
    • Works with multiple classes.
  • Disadvantages
    • Accuracy depends on selecting an appropriate probability distribution.
    • If the probability distribution can’t model the data, the accuracy will be poor.

Discriminative Models

Discriminative models directly model the posterior probability p(y∣x)p(y | \mathbf{x})p(y∣x).

Let’s start with the simplest discriminative model, the linear classifier.

Linear Classifier

For a linear classifier we have this “setup”. Observation (feature vectors) x∈RN\mathbf{x} \in \mathbb{R}^Nx∈RN. Class y∈{−1,+1}y \in \{-1, +1\}y∈{−1,+1}.

Our goal is to calculate a linear function of the feature vector x\mathbf{x}x,

f(x)=wTx+b=∑j=1Nwjxj+bf(\mathbf{x}) = \mathbf{w}^T \mathbf{x} + b = \sum_{j = 1}^N w_j x_j + bf(x)=wTx+b=j=1∑N​wj​xj​+b

where w∈RN\mathbf{w} \in \mathbb{R}^Nw∈RN is the weight vector and b∈Rb \in \mathbb{R}b∈R is the bias term.

From this value generated by the function, if f(x)>0f(\mathbf{x}) > 0f(x)>0, we predict class +1+1+1, otherwise we predict class −1-1−1, or just sign(f(x))sign(f(\mathbf{x}))sign(f(x)).

Geometric Interpretation

As the name suggests, the linear classifier seperates the feature space into 2 half-spaces.

Each space corresponds to feature values belonging to class +1+1+1 and class −1-1−1.

The class boundary is normal (orthogonal or perpendicular since we are in 2D) to w\mathbf{w}w. This boundary is also called the separating hyperplane.

Separating Hyperplane

In an NNN-dimensional feature space, the parameters are w∈RN\mathbf{w} \in \mathbb{R}^Nw∈RN.

The equation wTx+b=0\mathbf{w}^T \mathbf{x} + b = 0wTx+b=0 defines an N−1N-1N−1-dimensional linear surface.

  • For N=2N = 2N=2,w\mathbf{w}w defines a 1D line.
  • For N=3N = 3N=3, w\mathbf{w}w defines a 2D plane.
  • For N=nN = nN=n, w\mathbf{w}w defines a hyperplane.

Training a Linear Classifier

But how do we set these classifier parameters (w\mathbf{w}w, bbb)?

Simple, learn them from the data. But depending on which approach we take, we get different classifiers.

Let’s take a look at the probabilistic approach.

Logistic Regression

If we take the probabilistic route, we need to map the function value f(x)=wTx+bf(\mathbf{x}) = \mathbf{w}^T \mathbf{x} + bf(x)=wTx+b to a probability (value between 0 and 1).

Luckily, the sigmoid function does exactly this. It maps any real number to the interval [0,1][0, 1][0,1],

σ(z)=11+e−z, for z∈R\sigma(z) = \frac{1}{1 + e^{-z}}, \text{ for } z \in \mathbb{R}σ(z)=1+e−z1​, for z∈R

So, given a feature vector x\mathbf{x}x, the probability of a class is,

p(y=+1∣x)=σ(f(x))p(y=−1∣x)=1−σ(f(x))=σ(−f(x)) ∣ since σ(−z)=1−σ(z)p(y = +1 | \mathbf{x}) = \sigma(f(\mathbf{x})) \newline p(y = -1 | \mathbf{x}) = 1 - \sigma(f(\mathbf{x})) = \sigma(-f(\mathbf{x})) \ | \ \text{since } \sigma(-z) = 1 - \sigma(z)p(y=+1∣x)=σ(f(x))p(y=−1∣x)=1−σ(f(x))=σ(−f(x)) ∣ since σ(−z)=1−σ(z)

or simply,

p(y∣x)=σ(yf(x)).p(y | \mathbf{x}) = \sigma(y f(\mathbf{x})).p(y∣x)=σ(yf(x)).

So the important thing here is that we are direcrtly modeling the class posterior probability, not the class-conditional probability.

Learning the Parameters

Given the data set D={(x(i),y(i))}i=1M\mathcal{D} = \{(\mathbf{x}^{(i)}, y^{(i)})\}_{i = 1}^MD={(x(i),y(i))}i=1M​, we can learn the parameters (w\mathbf{w}w, bbb) using MLE.

Maximize the conditional log likelihood of the data D\mathcal{D}D,

(w⋆,b⋆)=arg⁡max⁡w,b∑i=1Mlog⁡p(y(i)∣x(i),w;b)=arg⁡max⁡w,b−1M∑i=1Mlog⁡(1+exp⁡(−y(i)(wTx(i)+b)))\begin{align*} (\mathbf{w}^\star, b^\star) &= \underset{\mathbf{w}, b}{\arg\max} \sum_{i = 1}^M \log p(y^{(i)} | \mathbf{x}^{(i)}, \mathbf{w}; b) \newline &= \underset{\mathbf{w}, b}{\arg\max} -\frac{1}{M} \sum_{i = 1}^M \log (1 + \exp(-y^{(i)} (\mathbf{w}^T \mathbf{x}^{(i)} + b))) \end{align*}(w⋆,b⋆)​=w,bargmax​i=1∑M​logp(y(i)∣x(i),w;b)=w,bargmax​−M1​i=1∑M​log(1+exp(−y(i)(wTx(i)+b)))​

To prevent overfitting, add a prior distribution on w\mathbf{w}w, for example, a Gaussian distribution with variance C/2C/2C/2.

Cross-Validation

We need to use cross-validation on training set to select the best value of CCC.

Run many experiments on the traning set to see which parameters work on different versions of the data.

First, partition the data into folds of training and validation data. Try a range of CCC values on each fold (as validation set). Simply, pick the value that works best over all folds.

Cross-Validation Algorithm

  • Procedure
    1. Select a range of CCC values to try.
    2. Repeat KKK rounds:
      1. Split the training set into training data and validation data.
      2. Learn a classifier for each value of CCC.
      3. Record the accuracy on the validation data for each CCC.
    3. Select the value of that has the highest average accuracy over all KKK rounds.
    4. Retrain the classifier using all data and the selected CCC.

Multiclass Classification

So far, we have only learned a classifier for 2 classes y∈{−1,+1}y \in \{-1, +1\}y∈{−1,+1}, also called a binary classifier.

For more than 2 classes, a good stragety is to simply, split the problem into several binary classifier problems.

One-vs-Rest

In this strategy, when we train, for each class, train a classifier for that class versus the other classes.

So for example, if we have 3 classes then we train 3 binary classifiers, 111 vs {2,3}\{2, 3\}{2,3}, 222 vs {1,3}\{1, 3\}{1,3}, and 333 vs {1,2}\{1, 2\}{1,2}.

When predicitng, calculate the probablity for each binary classifier. Select the class with the highest probablity.

Multiclass Logistic Regression

Another way to get a multiclass classifier is to define a multiclass objective function.

One weight vector wc\mathbf{w}_cwc​ for each class ccc. We omit the bias bcb_cbc​ for each class for notational simplicity.

Geometry of Multiclass Logistic Regression

In logistic regressino, it is explcitily designed to have a linear decision boundary in the binary case.

In the multiclass case, the decision boundary is piece-wise linear.

Logistic Regression VS. NB and LDA

  • Speed

    • Learning logistic regression requires iterative numerical optimization, which will be slower than NB and LDA.

  • Storage

    • Interpretability

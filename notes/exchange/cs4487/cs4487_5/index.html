The Regression Task

Definition of the regression task,

Given a feature vector x∈RN\mathbf{x} \in \mathbb{R}^Nx∈RN, predict its corresponding output vale y∈Ry \in \mathbb{R}y∈R.

Example: Curve Fitting

Given MMM points sampled from some underlying curve (assuming no measurement noise), the goal is to find that curve.

But let’s first think about this problem.

  • Would a small handful of points work well, or them or the better?
  • Given that the number of points are fixed, do their locations matter?
  • Given a set of MMM points, is curve fitting unique?
  • Lastly, how do we determine a “seemingly best” curve, compared to other possibilities?

I would say the answer to 1-3 are quite obvious, but the fourth one is the deal-breaker.

Answers

  • We always like more samples, if possible.
  • Samples need to be representative or informative.
  • Estimating continuous model from discrete data is an ill-posed problem and in most cases cannot be certain.
  • You need to have some criteria to choose your preferred model.

The Regression Learning Problem

Definition of the regression learning problem,

Given a data set of example pairs D={(x(i),y(i)),i=1,…,M}\mathcal{D} = \{(\mathbf{x}^{(i)}, y^{(i)}), i = 1, \ldots, M \}D={(x(i),y(i)),i=1,…,M} where x(i)∈RN\mathbf{x}^{(i)} \in \mathbb{R}^Nx(i)∈RN is a feature vector and y(i)∈Ry^{(i)} \in \mathbb{R}y(i)∈R is the output, learn a function f:RN↦Rf : \mathbb{R}^N \mapsto \mathbb{R}f:RN↦R that accurately predicts yyy for any feature vector x\mathbf{x}x.

More on Linear Regression

Take a closer look at the closed-form solution,

We call X†=(XTX)−1XT\mathbf{X}^{\dagger} = (\mathbf{X}^T \mathbf{X})^{-1} \mathbf{X}^TX†=(XTX)−1XT the pseudo-inverse (generalization of inverse to non-square matrix). See for a square invertible matrix X\mathbf{X}X, we have X†=X−1(XT)−1XT=X−1\mathbf{X}^{\dagger} = \mathbf{X}^{-1} (\mathbf{X}^T)^{-1} \mathbf{X}^T = \mathbf{X}^{-1}X†=X−1(XT)−1XT=X−1.

What if XTX∈R(N+1)×(N+1)\mathbf{X}^T \mathbf{X} \in \mathbb{R}^{(N+1) \times (N+1)}XTX∈R(N+1)×(N+1) is non-invertible?

  • If M<N+1M < N + 1M<N+1, add more data or enforce regularization.
  • If M≥N+1M \geq N + 1M≥N+1, remove redundant features or enforce regularization.

Regularized Linear Regression

As we have discussed previously, regularization is a technique to prevent overfitting. But also to obtain numerically more stable solutions (e.g., when XTX\mathbf{X}^T\mathbf{X}XTX is not invertible), and enforce the desired parameter space as a prior (e.g., select a subset of features that are good for prediction by enforcing the weights of irrelevant features to zero).

But the price we pay for having regularization is that the regularization term will bias the least squares estimate.

Ridge Regression

Let’s add regularization to OLS,

min⁡w12∥Xw−y∥22+α2∥w∥22\underset{\mathbf{w}}{\min} \frac{1}{2} \Vert \mathbf{X} \mathbf{w} - \mathbf{y} \Vert_2^2 + \frac{\alpha}{2} \Vert \mathbf{w} \Vert_2^2wmin​21​∥Xw−y∥22​+2α​∥w∥22​

The first term is the data-fit term, we sum the squared error of the prediction.

The second term is the regularization term, ∥w∥22=∑j=1N+1wj2\Vert \mathbf{w} \Vert_2^2 = \sum_{j=1}^{N+1} w_j^2∥w∥22​=∑j=1N+1​wj2​ penalizes large weights.

α\alphaα is the hyperparameter that controls the amount of “shrinkage”. Larger α\alphaα means more shrinkage, α=0\alpha = 0α=0 is OLS.

This also has a probabilistic interpretation,

p(w∣X;y,α)∝p(y∣w,X;σ2)p(w;α)p(\mathbf{w} | \mathbf{X}; \mathbf{y}, \alpha) \propto p(\mathbf{y} | \mathbf{w}, \mathbf{X}; \sigma^2) p(\mathbf{w}; \alpha)p(w∣X;y,α)∝p(y∣w,X;σ2)p(w;α)
But, Why?

Why should we penalize large weights in the first place?

Equivalently, why are smaller weights (i.e., weights that are close to zero) more preferred than larger weights?

  • Reason 1: Smaller weights are more robust to perturbations of input features.

    • Consider f1(x)=2x+0.1f_1(x) = 2x + 0.1f1​(x)=2x+0.1 and f2(x)=100x−98f_2(x) = 100x-98f2​(x)=100x−98.
    • If we add a small perturbation to x→x+0.1x \rightarrow x + 0.1x→x+0.1.
    • Then, Δf1(x)=0.2\Delta f_1(x) = 0.2Δf1​(x)=0.2 and Δf2(x)=10\Delta f_2(x) = 10Δf2​(x)=10.

  • Reason 2

    • There are better chances to zero out some input features xxx that are redundant or uninformative, leading to more accurate prediction of yyy.

Ridge Regression Closed-Form Solution

The closed-form solution for Ridge regression is,

w⋆=(XTX+αI)−1XTy\mathbf{w}^{\star} = (\mathbf{X}^T \mathbf{X} + \alpha \mathbf{I})^{-1} \mathbf{X}^T \mathbf{y}w⋆=(XTX+αI)−1XTy

The term “ridge regression” comes from the closed-form solution, where a “ridge” is added to the diagonal of XTX\mathbf{X}^T \mathbf{X}XTX.

Lasso Regression

With ridge regression, some weights are small but still non-zero. These are less important, but somehow still necessary.

To get a better shrinkage to zero, we can change the regularization term to encourge more weights to be zero.

The Lasso regression is defined as,

min⁡w12∥Xw−y∥22+α∥w∥1\underset{\mathbf{w}}{\min} \frac{1}{2} \Vert \mathbf{X} \mathbf{w} - \mathbf{y} \Vert_2^2 + \alpha \Vert \mathbf{w} \Vert_1wmin​21​∥Xw−y∥22​+α∥w∥1​

Least Absolute Shrinkage and Selection Operator, or LASSO.

Keeps the same data-fit term, but changes the regularization term. We take the sum of absolute weight values ∥w∥1=∑j=1N∣wj∣\Vert \mathbf{w} \Vert_1 = \sum_{j=1}^{N} |w_j|∥w∥1​=∑j=1N​∣wj​∣.

When a weight is close to zero, the regularization term will force it to be equal to zero.

This is a convex optimization problem, with no closed-form solution in general. However, it can be solved efficiently using an algorithm called least angle regression.

Why Shrinkage?

Under the orthogonal design (XTX=I)(\mathbf{X}^T \mathbf{X} = \mathbf{I})(XTX=I), we have,

Linear Regression Summary

  • OLS needs at least M≥N+1M \geq N + 1M≥N+1 data cases to learn a model with a NNN dimensional feature vector.
  • Ridge and Lasso work when XTX\mathbf{X}^T \mathbf{X}XTX is close to singular.
    • E.g., caused by co-linear features (xi≈axj+b)(x_i \approx ax_j + b)(xi​≈axj​+b).
  • MSE objective function for OLS, Ridge, and Lasso is sensitive to noise and outliers.
    • Regularization (Ridge and Lasso) can prevent very large weights, and reduce the possibility of overfitting to outliers.
  • All fail when output yyy non-linearly depends on input features x\mathbf{x}x.
